2020年：Google Cloud数据库年

2020-12-27 08:03:05

2020年是与众不同的一年，所有的意外都使基础企业技术成为人们关注的焦点。企业需要他们的数据库可靠，可扩展并且性能稳定。结果，加速了迁移计划，僵化的许可进一步失宠，并且变革性的应用程序开发加速了。根据Gartner Predicts的数据，即使在2019年也很明显，当时云数据库管理系统（DBMS）的收入为170亿美元，比2018年增长了54％。我们将渴望看到Gartner 2020年的报告，但从我们的角度来看，今年的增长显着加快。

我们相信，我们对开放性和灵活性的数据愿景已反映在今年的首个DBMS魔力象限中。 Gartner将Google Cloud评为2020年DBMS的领导者。

我们从各个行业的客户那里听说，这是他们开始或加强数据库现代化的一年。为了帮助他们实现关键任务目标，Google Cloud继续推出新产品和功能。这是今年的新事物和值得注意的事物。

现在可用于Cloud SQL的数据库迁移服务对企业而言，这可能是一个挑战。最近推出的无服务器数据库迁移服务（DMS）为我们的客户提供了独特的轻松，安全和可靠的体验，该服务为MySQL和PostgreSQL工作负载提供了高保真度，最少的停机时间迁移，并且旨在实现真正的云原生。宣布发布的我们的博客提供了更多信息，以及使您入门的步骤。

在云中管理的SQL Server企业公司经常告诉我们，迁移到SQL Server的Cloud SQL对他们的基础架构现代化和多云策略的更大目标有多重要。 SQL Server的Cloud SQL现在在全球范围内普遍可用，以帮助您保持SQL Server工作负载的运行。我们关于该主题的博客列出了开始迁移的五个步骤，完整迁移指南的链接以及有关更多详细信息的有用视频。

适用于Oracle数据库的Bare Metal Solution进入五个新的Google Cloud区域Bare Metal Solution使企业可以在Google Cloud Regional Extensions中运行特殊的工作负载（例如Oracle数据库），同时降低总体成本并降低与迁移相关的风险。去年，我们宣布了Bare Metal Solution在另外五个地区的可用性：弗吉尼亚州的阿什伯恩；法兰克福伦敦;加利福尼亚洛杉矶；和悉尼。今年，我们还推出了另外四个站点：阿姆斯特丹，圣保罗，新加坡和东京。

我们已经看到云迁移出现了一些明显的趋势。我们已经看到客户遵循我们所说的三个阶段的过程：迁移，即他们迁移大型商业数据库和开源数据库时；现代化，涉及从传统数据库到开源数据库的转变；和转型，构建下一代应用程序并开辟新的可能性。无论您在旅途中的何处，Google Cloud都致力于为您提供服务，最佳实践和工具生态系统的支持，以助您成功。

在制药和制药技术巨头McKesson上，团队选择了Cloud SQL对其遗留环境进行现代化。 3D打印和设计公司Makerbot分享了他们如何设计Google Cloud紧密集成的工具（包括Google Kubernetes Engine（GKE），Pub / Sub和Cloud SQL）以提供创新的自动缩放解决方案。

我们从大型零售商营销平台开发商Bluecore（该公司通过预测性数据模型进行营销活动）那里听说，他们如何转向Cloud SQL寻求一种完全托管的解决方案，该解决方案提供了营销活动创建功能，而不会降低零售品牌网站的速度。像Handshake（连接大学的平台的提供者）之类的客户也选择了Cloud SQL迁移。金融解决方案提供商Freedom Financial Network从Rackspace转到Cloud SQL，以满足不断增长的需求。

在Google Cloud Next '20：OnAir上，我们从ShareChat和《纽约时报》上听到了他们使用我们的云原生数据库所获得的成功。我们还从可汗学院（Khan Academy）那里听说，该学院使用Cloud Firestore来满足不断增长的在线学习需求。

如果Google Cloud发生区域性故障，您希望您的应用程序和数据库能够快速开始为另一个可用区域中的客户提供服务。今年，我们启动了Cloud SQL跨区域复制，可用于MySQL和PostgreSQL数据库引擎。我们与面临业务连续性挑战的Cloud SQL客户紧密合作，以简化体验，我们的博客介绍了入门方法，并介绍了美国职业棒球大联盟如何使用跨区域复制。此外，Cloud SQL为Postgres增加了承诺使用折扣，更多的维护控制，无服务器导出和时间点恢复。

去年秋天，我们宣布Cloud SQL现在支持MySQL8。您现在可以使用各种强大的新功能来提高生产率，例如即时DDL语句（例如ADD COLUMN），原子DDL，使用角色的特权收集，窗口函数，以及扩展的JSON语法。查看新功能的完整列表。

Cloud SQL数据库服务增加了PostgreSQL 13我们还为PostgreSQL 13提供了对Cloud SQL的支持，使您可以访问PostgreSQL的最新功能，同时让Cloud SQL处理繁重的操作。 PostgreSQL 13最近的全面性能改进包括增强的分区功能，增强的索引和清理效率以及更好的扩展监视。我们最近的博客提供了更多详细信息，更多功能和入门说明。

衡量用于Redis的Memorystore性能的工具Redis是一种流行的开源内存中数据存储，被用作数据库，缓存和消息代理。 Memorystore for Redis是Google Cloud的完全托管的Redis服务。 Memorystore最近添加了对Redis 5.0以及VPC服务控件，Redis Auth和TLS加密的支持。您将看到如何评估Memorystore for Redis的性能，以及内存管理，查询优化等性能调整的最佳做法。

Google Cloud Spanner是唯一具有无限规模，强大一致性和99.999％可用性的托管关系数据库。 （查看Spanner中的新增功能的更多详细信息。）我们在2020年宣布了Spanner的新企业功能，包括托管备份还原的常规可用性以及9个提供99.999％可用性的Spanner的新跨区域。 Spanner还引入了对新SQL功能的支持，包括查询优化器版本控制，外键，检查约束和生成的列。此外，Spanner还为C ++应用程序开发人员和本地仿真器引入了C ++客户端库，使您可以使用本地仿真器开发和测试应用程序，从而有助于降低应用程序开发成本。

Bigtable是我们完全托管的NoSQL数据库服务，现在提供托管备份以实现高业务连续性，并允许用户以最小的管理开销为工作负载添加数据保护。 Bigtable扩展了其对较小工作负载的支持，使您可以创建每个群集具有一个或两个节点的生产实例，而以前每个群集最少需要三个节点。

Firestore使移动和Web开发人员可以轻松构建应用程序，并添加了新功能，例如Rules Playground，可让您快速测试更新的Firebase Security规则。今年添加的Firestore Unity SDK使游戏开发人员更容易采用Firestore。此外，Firestore引入了C ++客户端库，并提供了更丰富的查询语言以及一系列新的运算符，包括“不包含”，“数组包含”，“不等于”，“小于”，“大于”和其他。
