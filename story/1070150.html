谷歌云提供了一个模型来修复谷歌的产品扼杀声誉

2021-07-28 03:07:15

我们之前已经多次写过这方面的文章：谷歌积极扼杀服务的声誉正在损害公司的品牌。谷歌推出的任何新产品都不再乐观，而是充斥着关于何时关闭的评论。这完全是 Google 自己造成的问题，但它又是另一个阻碍投资（时间、金钱或数据）最新的 Google 产品的障碍。 Google Stadia 今天面临的广泛怀疑就是一个很好的例子。一个有类似问题的谷歌部门是谷歌云平台，它要求公司和开发人员构建由谷歌云基础设施支持的产品或服务。与 Google 的其他部分一样，由于 API 很快被弃用，Cloud Platform 以不稳定着称，这使得托管在 Google 平台上的任何项目都需要不断更新以跟上最新的变化。不过，谷歌云希望通过新的“企业 API”名称来解决这个问题。企业 API 基本上有一个路线图，承诺某些 API 的稳定性。谷歌说，“负担在我们身上：我们的工作原则是，只要客户积极使用它，就不会删除任何功能（或以不向后兼容的方式进行更改）。如果弃用或破坏性更改是不可避免的，那么我们的负担是尽可能轻松地进行迁移。”如果 Google 确实需要更改 API，客户现在将收到至少一年的通知以及工具、文档和其他材料。 Google 继续说：“为了确保我们遵循这些原则，我们对 API 引入的任何更改都由产品和工程主管的集中委员会进行审查，并遵循严格的产品生命周期评估。”尽管是世界上最大的互联网公司之一，并且基本上定义了现代云基础设施的样子，但谷歌在云基础设施市场的表现并不好。 Canalys 以 7% 的市场份额将谷歌排在第三位，仅次于微软 Azure（19%）和市场领导者亚马逊网络服务（32%）。有传言（根据 The Information 的一份报告）谷歌云平台实际上面临着 2023 年的最后期限，以击败 AWS 和微软，否则将面临失去资金的风险。前谷歌员工史蒂夫耶格去年在一篇题为“亲爱的谷歌云：你的弃用政策正在杀死你”的帖子中阐述了谷歌云平台的问题。谷歌的声明似乎触及了这篇文章所强调的大部分内容，比如缺乏文档和支持，API 升级的无休止的跑步机，以及谷歌云对向后兼容性的普遍忽视。 Yegge 认为，像 Windows、Java 和 Android 等成功的平台（Yegge 说这个群体与更大的 Google 文化隔离开来）在很大程度上归功于他们对平台稳定性的承诺，并且从不破坏任何东西。 AWS 是市场领导者，部分原因是它被认为比谷歌云平台稳定得多。 Protocol 报道称，Google 副总裁 Kripa Krishnan 在公告期间实际上被问到她是否熟悉由 Cody Ogden 运营的 Killed By Google 网站和 Twitter 帐户。报告称，克里希南“忍不住笑了”，并说，“从互联网上的许多来源来看，我们很明显表现不佳。”谷歌云平台对谷歌声誉的认识、限制对客户干扰的措施以及它对哪些产品比其他产品更稳定的沟通，感觉就像是公司其他部门的榜样。许多 Google 产品目前都担心它们会被关闭，而且通常这足以让人们寻求其他替代方案。解决这个问题的主要方法只是缓解——即，停止关闭如此多的东西，一直。但紧随其后的是沟通——只需告诉人们您对未来支持的计划是什么。谷歌似乎没有问题为其硬件设备上的软件提供公共路线图。 Pixel 手机和 Chromebook 都有其软件的公开支持声明，显示了设备可以依赖支持的最低（这是最低限度！）日期。例如，我们知道 Pixel 5 至少会在 2023 年 10 月之前继续接收更新。谷歌无法立即解决其杀死产品和服务的声誉，但这样的沟通可以帮助用户和公司缓解一些犹豫在投资 Google 产品时越来越有感觉。如果你长期不打算杀死一个产品，那就说出来！告诉 Google 用户和公司合作伙伴哪些产品是稳定的，哪些是夜间实验。

当然，要使其发挥作用，Google 必须切实遵守其做出的任何公开承诺，以便人们相信它会兑现承诺。最近，该公司没有这样做。它承诺为 Android Things（Android 的物联网版本）提供三年支持。它没有兑现这一承诺。相反，谷歌仅在一年后就结束了操作系统更新。如果谷歌真的想修复其不稳定的声誉，它需要一个多年的旅程。
