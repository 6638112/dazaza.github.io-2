平台即服务初创公司 Porter 旨在成为部署、管理基于云的应用程序的首选平台

2021-07-30 21:48:42

他们意识到，尽管有所有好处，Rhee 告诉 TechCrunch，该技术已经存在，但用户必须管理托管解决方案的复杂性，并承担与大型 DevOps 团队相关的成本。他们决定在外部构建一个解决方案，并通过 Y Combinator 的 2020 年夏季批次，在那里他们发现其他初创公司也在尝试做同样的事情。今天，Porter 宣布从 Venrock、Translink Capital、Soma Capital 和几位天使投资人那里获得 150 万美元的种子资金。它的目标是构建一个平台即服务，任何团队都可以使用它来管理自己云中的应用程序，本质上通过类似 Heroku 的体验提供 Kubernetes 的全部灵活性。为什么是 Heroku？它是开发人员习惯的托管平台，不仅是小公司，还有后期公司。 Shim 补充说，当他们想要迁移到 Amazon Web Services、Google Cloud 或 DigitalOcean 时，Porter 将成为他们的桥梁。然而，虽然 Heroku 仍然很受欢迎，但两人表示公司认为该平台已经过时了，因为它在技术方面仍然停滞不前。 Rhee 说，每年，由于技术限制和成本，公司都会离开该平台。他说，Porter 押注的很大一部分不是向用户收取托管费用，其成本是纯粹的 SaaS 产品。他们不希望成为经销商，因此公司可以使用自己的云，但 Porter 将提供自动化，用户可以使用他们的 AWS 和 GCP 积分付款，这为他们提供了灵活性。 Shim 补充说，一个常见的模式是迁移到 Kubernetes，但“我们谈论的 zinger”是，如果 Heroku 是在 2021 年构建的，它将是基于 Kubernetes 构建的。

作为桥梁，该公司将利用新资金增加其工程带宽，目标是“成为所有初创公司的事实上的标准”。希姆说。 Venrock 合伙人 Ethan Batraski 表示，Porter 的平台于 2 月上线，并在六个月内成为 GitHub 上增长速度第六快的开源平台下载。他通过 YC 认识了这家公司，“对 Rhee 和 Shim 的愿景印象非常深刻。 “Heroku 有 100,000 名开发人员，但我相信它已经停滞不前，”Batraski 补充道。 “波特的平台上已经有 100 家初创公司。他们看到的增长——四到五倍——是你在这个阶段想要看到的。”他的公司长期以来一直专注于数据基础设施，并且看到堆栈变得越来越复杂，他说“与此同时，越来越多的开发人员希望在一周内构建一个应用程序，并将其扩展到数百万用户，但这需要人力资源.有了 Kubernetes，它可以将每个人都变成专家开发人员，而他们却不知道，”他补充道。 “Heroku 有 100,000 名开发人员，但我相信它已经停滞不前，”Batraski 补充道。 “波特的平台上已经有 100 家初创公司。他们看到的增长——四到五倍——是你在这个阶段想要看到的。”
