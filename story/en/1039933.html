Mundane: Rust cryptography library that is difficult to misuse
世俗：难以滥用的Rust密码库

2020-12-16 04:36:48

Mundane is a Rust cryptography library backed by BoringSSL that is difficultto misuse, ergonomic, and performant (in that order).

Mundane是由BoringSSL支持的Rust密码术库，它很难被滥用，符合人体工程学且性能优异（按此顺序）。

Mundane vendors a copy of the BoringSSL source, so BoringSSL does not need to beinstalled locally in order to build. However, the BoringSSL build system has thefollowing dependencies:

世俗的供应商提供了BoringSSL源的副本，因此无需在本地安装BoringSSL即可进行构建。但是，BoringSSL构建系统具有以下依赖性：

Perl 5.6.1 or later. See BoringSSL's buildinstructionsfor what to do if CMake fails to find Perl on your system.

Perl 5.6.1或更高版本。有关CMake无法在系统上找到Perl的操作，请参见BoringSSL的构建说明。

Either Make or Ninja. Ninja is preferable, as itmakes compilation significantly faster; if both are present, Ninja will beused. On Windows, Ninja is required.

Make或Ninja。 Ninja是更可取的，因为它使编译速度大大加快。如果两者都存在，忍者将被使用。在Windows上，需要Ninja。

To build the x86 and x86_64 assembly, your assembler must support AVX2instructions and MOVBE. If using GNU binutils, you must have 2.22 or later.

要构建x86和x86_64程序集，您的汇编程序必须支持AVX2指令和MOVBE。如果使用GNU binutils，则必须具有2.22或更高版本。

In order to avoid errors at link time due to conflicting symbols, we buildBoringSSL with a custom prefix for all of its symbols which is based on the nameand version of this crate. That way, even if multiple different versions ofMundane are present in the same dependency graph, none of the symbols from oneversion's BoringSSL will conflict with the symbols from another version'sBoringSSL.

为了避免在链接时由于符号冲突而产生错误，我们使用其板条箱的名称和版本为其所有符号构建了带有自定义前缀的BoringSSL。这样，即使同一依赖关系图中存在多个不同版本的Mundane，oneversion BoringSSL的任何符号也不会与另一版本的BoingSSL的符号冲突。

Mundane supports being built on and for Linux and Mac. Windows support is underdevelopment. Cross-compilation is not supported.

Mundane支持在Linux和Mac上以及针对Linux和Mac构建。 Windows支持尚不完善。不支持交叉编译。
