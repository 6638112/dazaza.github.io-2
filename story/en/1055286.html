Gimp 2.10.24 Released
GIMP 2.10.24发布

2021-04-01 19:40:10

GIMP 2.10.24 is mostly a bug fix release, which once again comes mostlywith file format support improvements.

GIMP 2.10.24主要是一个错误修复释放，它再次出现了文件格式的支持改进。

New “ Negative Darkroom” operation to simulate enlargement prints from scans of photographic negatives.

新的“负暗室”操作来模拟摄影底片扫描的扩大印刷。

Though there are various fixes in core code, this change is probably themost worth mentionning. Ever since GIMP 2.10.14, we are now able to seethe out-of-canvasarea.Consequently, many features can now work outside the canvas, yet not allfeatures yet. This change is the continuation of this work, allowing youto snap various tools to guides, grids or vectors, even outside the canvas.

虽然核心代码中有各种修复，但这种变化可能是值得一提的。自GIMP 2.10.14以来，我们现在能够脱掉帆布。当然，许多功能现在可以在画布外工作，但尚未解决所有特征。这种变化是这项工作的继续，允许您将各种工具捕捉到引导，网格或向量，即使在画布之外。

A lot of work has been going on in the metadata area, mostlyconsolidating our support and fixing many issues.

元数据区域已经进行了大量工作，大多数是我们的支持和解决许多问题。

The metadata viewer and editor also received a lot of love, making themmore robust to various edge cases, such as duplicate tags, but alsomapping equivalency between similar IPTC and XMP tags, better encodinghandling, and so on.

元数据查看器和编辑器还接收了很多爱，使其成为各种边缘案例的强大，例如重复标记，但在类似的IPTC和XMP标签之间，更好的编码负责等，均可均匀。

The GPS data is also handled a bit better with more precision, tooltipsand better formatting.

GPS数据也可以更好地处理，更精确，工具技术更好地格式化。

There are still a lot of improvements to be made in the metadata areathough we are probably on the right path. This part of the developmentis not as visible as other, yet this is very time-consuming andthankless groundwork while being useful to anyone who needs goodmetadata support so we’d like to thank Jacob Boerema who has workedtirelessly on this for months.

在元数据中仍有很大的改进，我们可能在正确的道路上。这一部分的发展不像其他人那样可见，但这是非常耗时的，并且对于任何需要GoodometAdata支持的人有用，因此我们要感谢雅各布的雅各布Boerema几个月。

A fun story which started with a conference by Adam Cox of LouisianaState University aboutusing GIMP for enhancing historic maps, with the issue that GeoTIFFmetadata tags were lost and made the workflow a bit more cumbersome.

一个有趣的故事，它始于Louisianastate大学的Adam Cox大会关于加强历史地图的Gimp，这是GeoTiffmetadata标签丢失并使工作流程更加繁琐的问题。

It prompted a bug report then later a patch by the passing contributorRuthra Kumar and a review by the core team. All this within 2 months.

它提示出一个错误报告，然后通过COMPORTURURTRA KUMAR的补丁以及核心团队的评论。所有这一切在2个月内。

And now GIMP is able to import and export back the GeoTIFF tags. Notethat no semantic logics is implemented, i.e. that GIMP can only exportwhat it imported (the checkbox will only be sensitive on export if therewas GeoTIFF metadata on import). It will not tweak the metadata contentsfor you. In particular since it contains georeferencing data, some typeof image transform could make the data meaningless. This is up to you toknow what the data references and how to keep its meaning.

现在GIMP能够导入和出口地理率标签。 Notethat没有实现语义逻辑，即，Gimp只能导出它导入的导出（复选框只会在导入的地理仪元数据时才能敏感）。它不会调整元数据内容。特别是由于它包含地理转移数据，某些类型的图像变换可能会使数据毫无意义。这取决于您才能知道数据引用以及如何保持其含义。

This nice little story shows once again a power of Free Software, whichis before all a software made by yourself. Anyone who contributes ispart of the GIMP team! 🤗

这个漂亮的小故事再次出现一次自由软件的力量，在您自己制作的所有软件之前。任何贡献ISPart的GIMP团队的人！ 🤗

Note: the sharpest mind may have realized the feature was available inthe development release 2.99.4. Yet we add the description for 2.10.24because this is the first stable release featuring GeoTIFF support.

注意：最敏锐的思维可能已经意识到该功能可用Inthe开发版本2.99.4。然而，我们添加了2.10.24的描述，因为这是第一个具有地理员支持的稳定释放。

TIFF got various improvements when handling multi-page files, but also many edge cases, such as 2 or 4-bit TIFF images, opening some types of non-conformant TIFF files and so on.

处理多页文件时，TIFF在处理多页文件时进行了各种改进，也有许多边缘案例，例如2或4位TIFF图像，打开某些类型的非符合性TIFF文件等。

HEIF got some visually lossless export support when libheif 1.10 or later is used. We also detect separately HEIC and AVIF support at runtime, allowing to build the plug-in with only support of one encoding.

当使用Libheif 1.10或更高版本时，Heif在视觉上导致导出支持。我们还在运行时检测单独的HEIC和AVIF支持，允许仅支持一个编码的插件。

PNG now ignores useless layer offset of 0, a metadata which some third-party software are always storing, hence getting rid of unecessary dialog prompts.

PNG现在忽略了无用的层偏移0，这是一些第三方软件始终存储的元数据，从而摆脱不必要的对话框。

BMP in more bit depth can now be loaded, in particular 24bpp BMP images; moreover GIMP is now able to rescue some non-conformant BMP with wrong compression noted in header.

现在可以加载更多位深度的BMP，特别是24BPP BMP图像;此外，GIMP现在能够拯救一些非符合性BMP，以在标题中注明的错误压缩。

PDF import now proposes an option to reverse order of layers (same as we already had on export) and now support fractional DPI import.

PDF Import现在提出了一种选择相反的图层（与导出相同），现在支持分数DPI导入。

DDS in BC5 format benefited from some fixes. Moreover as we are able to detect some images with errors previously created by GIMP, the software will also automatically fix these errors upon loading them.

BC5格式中的DDS受益于某些修复程序。此外，由于我们能够通过GIMP创建的错误检测一些图像，因此该软件还将在加载它们时自动修复这些错误。

Raw image formats are still forwarded through featureful raw developers such as darktable or RawTherapee. The former is undergoing some API changes, and while darktable 3.6 is not even out yet, GIMP already has support for this upcoming version. Therefore GIMP 2.10.24 will work with future darktable.

RAW IMAGE格式仍然通过暗形或Rawtherapee等配合原始开发人员转发。前者正在进行一些API变化，而暗形3.6甚至没有出发，GIMP已经支持即将推出的版本。因此，GIMP 2.10.24将与未来的暗无线电合作。

GIMP is now available in one more language: Kabyle. This is still anearly translation as only 18% of the stable branch istranslated so far(and 32% of the development branch!) yet we can already thank these newtranslators to bring GIMP to even more people.

GIMP现在有一种语言：Kabyle。这仍然是一个公正的翻译，到目前为止只有18％的稳定分支机构（以及32％的开发分支机构！）我们已经感谢这些新手家将GIMP带到更多的人。

Translators are also contributors doing an incredible work, even thoughtheir work doesn’t always get the visibility and acknowledgement theydeserve. Thanks to all of them!

翻译人员也是贡献者做出了令人难以置信的工作，即使他的工作并不总是获得能见度和致谢他们的能力。谢谢他们所有人！

Our pixel encoding and color space conversion engine, babl 0.1.86, nowsupports creating babl space from input-class ICC profiles and improvesthread safety.

我们的像素编码和色彩空间转换引擎Babl 0.1.86，毫不床头支持从输入级ICC配置文件创建Babl空间和InflestSthread安全。

On its side, GEGL 0.4.30 made improvements to its test suite and tothe following operations: jpg-load, png-load, tiff-load, rgbe-load, color-reduction, fattal02 and paint-select. Thislater operation in particular was introduced for the new Paint Selecttoolso we will have the opportunity to talk about the improvements on thenext development release news.

在其侧面，GEGL 0.4.30对其测试套件进行了改进，并在操作之后进行了改进：JPG-LOAD，PNG-LOAD，TIFF加载，RGBE负载，颜色还原，Fattal02和PAINT-SELECT。特别是为新的PaintSeedToolso引入了本发明的操作，我们将有机会谈谈此开发释放新闻的改进。

Additionally to the other improvements, a new very interesting operationappears in GEGL, contributed by Jonny Robbie: negative-darkroom.

此外，还有其他改进，由Jonny Robbie：负暗室的新的GEGL中出现了一个新的非常有趣的杂志。

This operation is for artists who use hybrid workflow technique ofanalog photography. After scanning a developed negative, this operationis used to invert the scan to create a positive image by simulating thelight behaviour of darkroom enlarger and common photographic papers.

这项操作适用于使用混合工作流技术的艺术家。在扫描显影否定后，通过模拟暗室放大和常见摄影文件的大学行为来反转扫描来创建正面图像。

As all GEGL operations are automatically detected and made available by GIMP, this new operation can be used in GIMP 2.10.24 through the generic GEGL tool (menu Tools > GEGL Operation… then select “ NegativeDarkroom (negative-darkroom)” in the dropdown list).

由于所有GEGL操作都是由GIMP自动检测到的，这款新操作可用于GIMP 2.10.24通过通用GEGL工具（菜单工具和gt; GEGL操作......然后在下拉列表中选择“NegativedArchoom（负暗室）”列表）。

Creating a custom dedicated dialog for this operation has been raisedand may happen in an further version of GIMP to even more improve theusage and experience.

为此操作创建自定义专用对话框已被举报人，可能会发生在GIMP的另一个版本中，以更高，以更高的函件和体验。

Meanwhile babl minimum requirement in GEGL has been downgraded to 0.1.78(same as in GIMP) because newer versions require too recent mesonbuild tool, which is unfortunately still not available on somedistributions. In order not to prevent people from benefiting from anewer version of GEGL and GIMP, we refrain on purpose to bump theminimum requirement for a bit even though we highly encourage everypackager to use the last version of babl when possible. Many fixes andimprovements were also made available in recent versions.

同时，GEGL的BABL最低要求已将其降级为0.1.78（与GIMP相同），因为较新的版本需要Then最近的MesonBuild工具，而且尚不讨论一条内容。为了不防止人们从ANEWER版本的GEGL和GIMP中受益，我们避免了一个目的，即使我们高度鼓励everymacker在可能的情况下使用最后版本的BAB1，也可以获得一点的最小要求。最近的版本也提供了许多修复的安提维修。

The Linux flatpak has already been published so that anyone who installed it previously should have an update proposed by their software manager (or from terminal: flatpak update org.gimp.GIMP).

Linux FlatPak已经发布，以便安装之前的任何人都应该由他们的软件管理器（或从终端：FLOTPAK更新org.gimp.gimp.gimp.gimp）进行更新。

Note: our flatpak now supports only x86-64 and AArch64 (i.e. the 64-bit variants of the x86 and ARM architectures). In particular i386 (32-bit x86) had been dropped quite some time ago by the Freedesktop runtime we depend on. This is now the ARM (32-bit) support which has been dropped (even though 32-bit hardware is still being released or often 64-bit board computers are sold with a 32-bit OS). We tried to hold back a bit, for more than 6 months even, but now that the older runtime we used is unsupported, updating is the only sane choice.

注意：我们的FLATPAK现在只支持X86-64和AARCH64（即X86和ARM架构的64位变体）。特别是i386（32位x86）很久以前被替代的Freedesktop运行时已经掉了一段时间。这是现在已删除的ARM（32位）支持（即使仍然释放32位硬件或通常使用32位OS出售64位硬件）。我们试图重新遏制一下，甚至超过6个月，但现在我们使用的旧运行时不受支持，更新是唯一的理智选择。

For the record, our i386 flatpak is therefore stuck at GIMP 2.10.14 and our ARM flatpak is stuck at GIMP 2.10.22 with a few thousands downloads for this last version of GIMP i386 and a bit more than 400 for the last version of GIMP ARM.

因此，我们的I386 FlatPak陷入了GIMP 2.10.14，我们的ARM FLANPAK陷入了GIMP 2.10.22，在最后一个版本的GIMP I386下载了几千个下载，最后一个版本的GIMP超过400手臂。

The Windows installer is already available. Most mirrors have picked it up, but some still haven’t. So if the download fails, just try to click the Download button again.

Windows Installer已经可用。大多数镜子都拿起了它，但有些仍然没有。因此，如果下载失败，请尝试再次单击下载按钮。

The macOS DMG package will be published in the next few days once our packager can make the time.

一旦我们的包装机可以花时间，宏将在接下来的几天内发布。

The development continues very strong on the development branch and wecan clearly see the shift towards more work on GIMP 3 as 2.10.x releasebecome more about robustness and less about new features (though westill continue to backport features when it can be done without too muchadditional work).We will give more details on this side of development when we willrelease the upcoming 2.99.6 development version.

发展分支机构的发展仍然很强劲，Wecan清楚地看到了更多关于GIMP 3的工作的转变为2.10.x referenceBecome更多关于鲁棒性和更少的关于新功能（尽管Westill在没有太多的情况下完成时，但在没有太多的工作）。我们将在将来的2.99.6开发版本中提供有关开发方面的更多细节。

Finally, please don’t forget you can donate to the project and personallyfund several GIMP developers, as a way togive back and accelerate the development of GIMP.

最后，请不要忘记你可以捐赠给项目并个人交往几个GIMP开发人员，作为一种方式，作为一种方式来回归并加快GIMP的发展。
