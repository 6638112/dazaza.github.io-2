Glow：带TUI和加密云存储的终端降价阅读器

2020-10-17 23:09:41

Glow is a terminal based markdown reader designed from the ground up to bringout the beauty—and power—of the CLI.

Glow是一款基于终端的降价阅读器，旨在展现CLI的美感和强大功能。

Use it to discover markdown files, read documentation directly on the commandline and stash markdown files to your own private collection so you can readthem anywhere. Glow will find local markdown files in subdirectories or a localGit repository.

使用它可以发现标记文件，直接读取命令行上的文档，并将标记文件隐藏到您自己的私有集合中，这样您就可以在任何地方读取它们。Glow将在子目录或localGit存储库中找到本地标记文件。

By the way, all data stashed is encrypted end-to-end: only you can decrypt it.More on that below.

顺便说一句，所有隐藏的数据都是端到端加密的：只有您才能解密。

Simply run glow without arguments to start the textual user iterface andbrowse local and stashed markdown. Glow will find local markdown files it thecurrent directory and below or, if you’re in a Git repository, Glow will searchthe repo.

只需运行不带参数的Glow即可启动文本用户界面，并浏览本地和隐藏的标记。Glow将在当前目录下找到本地标记文件，或者，如果你在Git库中，Glow将搜索repo。

Glow works with the Charm Cloud to allow you to store any markdown files inyour own private collection. You can stash a local document from the Glow TUI bypressing s.

Glow与Charm Cloud配合使用，允许您将任何标记文件存储在您自己的私人收藏中。您可以通过按s来隐藏发光TUI中的本地文档。

Then, when you run glow without arguments will you can browse through yourstashed documents. This is a great way to keep track of things that you need toreference often.

Stashing is private, its contents will not be exposed publicly, and it'sencrypted end-to-end. More on encryption below.

Encryption works by issuing symmetric keys (basically a generated password) andencrypting it with the local SSH public key generated by the open-source charm library. That encrypted key is then sent up to our server.We can’t read it since we don’t have your private key. When you want to decryptsomething or view your stash, that key is downloaded from our server anddecrypted locally using the SSH private key. When you link accounts, thesymmetric key is encrypted for each new public key. This happens on yourmachine and not our server, so we never see any unencrypted data.

加密的工作方式是发布对称密钥(基本上是生成的密码)，并使用开源Charm库生成的本地SSH公钥对其进行加密。然后将该加密密钥发送到我们的服务器。我们无法读取它，因为我们没有您的私钥。当您想要解密某些东西或查看您的藏品时，该密钥将从我们的服务器下载并使用SSH私钥在本地解密。当您链接帐户时，会为每个新的公钥加密对称密钥。这发生在您的机器上，而不是我们的服务器上，因此我们永远不会看到任何未加密的数据。
