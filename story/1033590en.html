level 1 Whether int x = x * 42; is valid or not (or what its semantics are if it is valid), has nothing to do with whether a language's grammar is left recursive or not. In fact any left recursive grammar can be rewritten to not be left recursive without changing what is and isn't syntactically valid.

第一级，int x=x*42；是否有效(或者如果有效，它的语义是什么)与一种语言的语法是否保持递归无关。事实上，任何左递归语法都可以重写成不是左递归的，而不会改变什么是语法有效的，什么是不是语法有效的。

In any language that allows variable declarations of the form int x = expression; and that allows x * 42 as a valid expression, int x = x * 42; will be syntactically valid. However, in many languages it will be a scoping error that would be caught in the semantic analysis phase. That depends on the scoping rules of the language though, not on the grammar.

在任何语言中，如果允许变量声明的形式为int x=Expression；并且允许x*42作为有效的表达式，则int x=x*42；在语法上是有效的。然而，在许多语言中，这将是在语义分析阶段捕捉到的作用域错误。不过，这取决于语言的作用域规则，而不是语法。

In the specific case of C, int x = x * 42; is not a scoping error ( x is in scope on the right side of =), but invokes undefined behavior at runtime because x is being used uninitialized. The commonly used C compilers will issue a warning in this case (assuming you enabled warnings, that is), so yours probably should too.

在C的特定情况下，int x=x*42；不是作用域错误(x在=右侧的作用域中)，而是在运行时调用未定义的行为，因为x在未初始化的情况下使用。在这种情况下，常用的C编译器将发出警告(假设您启用了警告)，因此您的C编译器可能也应该发出警告。

level 1 There is a general technique to stop recursion (assuming you want to do that of course): a historical trace or trail. Basically you add an auxiliary argument to all participating functions which contains a purely functional list of all the cases you have already handled. Even new case gets cons'ed onto the list before doing anything, unless it is already on the list, in which case you have started to repeat yourself and can stop the recursion.

级别1有一种停止递归的通用技术(当然，假设您想这样做)：历史跟踪或跟踪。基本上，您可以向所有参与函数添加一个辅助参数，该参数包含您已经处理过的所有案例的纯函数列表。即使是新的案例也会在执行任何操作之前被添加到列表中，除非它已经在列表中了，在这种情况下，您已经开始重复自己，并且可以停止递归。

There is a more efficient way to do this if you design you data structures to handle this case in advance and are willing to use mutation: you can just flip a flag on the nodes of the data structure indicating it has already been visited. In fact this is the basic way mark-sweep garbage collector work.

如果您事先设计数据结构来处理这种情况，并且愿意使用突变，那么有一种更有效的方法来实现这一点：您只需在数据结构的节点上翻转一个标志，表明它已经被访问。事实上，这是标记清除垃圾收集器的基本工作方式。

In more specialised cases, more efficient solutions may exist. I use trails extensively in my compiler. The timing of and place where you add to and check the trail is critical and very hard to get right.

在更特殊的情况下，可能存在更有效的解决方案。我在我的编译器中广泛使用跟踪。添加和检查踪迹的时机和地点非常关键，而且很难正确处理。
