defmodule FooBar do def foo(a) do if a < 0 do bar(a, -1) else bar(a, 1) end end defp bar(a, b) do IO.inspect(a * b) endend

Defmodule FooBar do def foo(A)do if a<；0 do bar(a，-1)Else bar(a，1)end end defp bar(a，b)do IO.check(a*b)endend。

I like Elixir, but I think most of the time it just looks like functional Ruby. I want to make this code look like Elixir.

我喜欢长生不老药，但我认为大多数时候它看起来就像功能强大的Ruby。我想让这个代码看起来像长生不老药。

First, this code is lacking every Elixir developer's best friend: |>. Let's add some |>s.

首先，这段代码缺少所有ELEXIR开发人员最好的朋友：|>；。让我们添加一些|>；。

defmodule FooBar do def foo(a) do if a < 0 do a |> bar(-1) else a |> bar(1) end end defp bar(a, b) do (a * b) |> IO.inspect() endend

Defmodule FooBar do def foo(A)do if a<；0 do a|>bar(-1)否则a|>bar(1)end end defp bar(a，b)do(a*b)|>；IO.check()endend。

Meh. It's definitely better, but I mean, that's only 3 |>s. I want more |>s.

嗯。它绝对更好，但我的意思是，那只是3个，我想要更多。

defmodule FooBar do def foo(a) do (a < 0) |> if do a |> bar(-1) else a |> bar(1) end end defp bar(a, b) do (a * b) |> IO.inspect() endend

Defmodule FooBar do def Foo(A)do(A)do(a<；0)|&>；if do a|&>bar(-1)否则a|&>bar(1)end end defp bar(a，b)do(a*b)|>；IO.check()endend。

defmodule FooBar do def foo(a) do a |> Kernel.<(0) |> if do a |> bar(-1) else a |> bar(1) end end defp bar(a, b) do a |> Kernel.*(b) |> IO.inspect() endend

Defmodule FooBar do def foo(A)do|&>；Kernel。<；(0)|>；if do|&>bar(-1)否则|&>bar(1)end end defp bar(a，b)do a|&>；Kernel。*(B)|&>IO.check()endend。

The Elixir docs say defmodule is just a macro. Does that mean I can just |> into defmodule?

药剂医生说defmodule只是一个宏。这是不是意味着我可以直接|&>进入demodule？

FooBar |> defmodule do def foo(a) do a |> Kernel.<(0) |> if do a |> bar(-1) else a |> bar(1) end end defp bar(a, b) do a |> Kernel.*(b) |> IO.inspect() endend

FooBar|&>；defmodule do def Foo(A)do|&>；Kernel.<；(0)|>；if do|&>bar(-1)否则a|&>；bar(1)end defp bar(a，b)do a|&>；Kernel.*(B)|>；IO.check()end。

FooBar |> defmodule do a |> foo() |> def do a |> Kernel.<(0) |> if do a |> bar(-1) else a |> bar(1) end end a |> bar(b) |> defp do a |> Kernel.*(b) |> IO.inspect() endend

FooBar|&>defmodule执行|>；foo()|&>；def执行|&>；内核。<；(0)|>；如果执行|&>bar(-1)，则|&>bar(1)结束|&>bar(B)|&>defp执行|&>；内核。*(B)|>；IO.spect()end.。

We're getting somewhere, but something still doesn't feel right. This module really isn't doing much, so maybe it should not be that long? Also, I think we need more :. Atoms are very Elixir-y, so let's do more of that:

我们正在取得进展，但还是感觉有些不对劲。这个模块真的做不了什么，所以也许它不应该那么长？此外，我认为我们需要更多：。原子是非常灵丹妙药的，所以让我们做更多的事情吧：
