覆盆子PI 400-100美元以下的台式电脑，你不知道你需要它

2020-11-03 06:15:34

Late Friday afternoon, I got an exciting SMS notification—my review sample of the new Raspberry Pi 400 had arrived. I learned of the new Pi model last week while interviewing Raspberry Pi founder Eben Upton and Canonical desktop engineering director Martin Wimpress about Ubuntu 20.10's newly improved desktop support for the Pi hardware family.

周五下午晚些时候，我收到了一条令人兴奋的短信通知--我的新款覆盆子PI 400的点评样品已经到了。上周，我在采访Raspberry Pi创始人埃本·厄普顿(Eben Upton)和Canonical桌面工程总监马丁·温普拉斯(Martin Wimpress)时了解到了新的PI模型，内容是关于Ubuntu20.10；对PI硬件系列的新改进桌面支持。

In brief, the Pi 400 is a slightly faster version of the 4GiB Pi 4 which ships preassembled in a small, wedge-shaped chassis with integrated keyboard. The new model directly targets desktop replacement use and can be purchased solo for $70 or as a full kit (as seen above) for $100.

简而言之，PI 400是4GiB PI 4的稍微快一点的版本，4GiB PI 4预装在一个带有集成键盘的小楔形机箱中。这款新机型直接针对台式机更换用途，可以单独购买，价格为70美元，也可以作为全套套件(如上所示)购买，价格为100美元。

The new form factor—which has apparently been in the works ever since the introduction of the official Raspberry Pi keyboard—addresses and enthusiastically supports the Pi 4's growing use case as a replacement or alternative for the traditional desktop PC. Upton told Ars that the Pi 400 is about 20 percent faster than the Pi 4; it has largely the same components under the hood but on a differently laid-out board, and its BCM2711 CPU is clocked a touch higher than the BCM2711 in the Pi 4.

自Raspberry Pi官方键盘推出以来，新的外形规格显然就一直在研发中，它满足并热情支持PI 4作为传统台式PC的替代品或替代品的日益增长的使用案例。厄普顿告诉ARS，PI 400比PI 4快约20%；它的引擎盖下有基本相同的组件，但在布局不同的电路板上，其BCM2711 CPU的时钟略高于PI 4中的BCM2711。

Unboxed and plugged in, the Pi 400 is functional but not particularly lovely. The integrated keyboard means one fewer cable to deal with, and I found the cables unusually likely to snarl and look a bit feral. They are both stiffer and shorter than I'd prefer in an ideal world, making it difficult-to-impossible to end up with a setup that doesn't look like a rat's nest. The red cable for the mouse clashes pretty violently with the off-white cables for USB-C power and micro-HDMI out, which doesn't help any.

PI400没有包装，插上电源，功能很好，但不是特别可爱。集成键盘意味着要处理的线缆减少了一条，我发现线缆很可能会缠绕在一起，看起来有点野性。它们都比理想世界中的我喜欢的更硬，也更短，所以很难甚至不可能最终得到一个看起来不像老鼠窝的设置。鼠标的红色线缆与USB-C电源和微型HDMI输出的灰白色线缆发生了相当激烈的冲突，这对任何事情都没有帮助。

That said, it's important to remember that the entire kit retails for $100. Within the limits of the Pi 400's very generous price, it's not really fair to complain too hard about a few aesthetic gaffes here and there! Consumers with a few extra dollars to spend might want to consider replacing the Pi 400's mouse with something a bit more functional, though... and a full-sized keyboard might not be a bad idea while you're at it.

这就是说，重要的是要记住，整个套件的零售价是100美元。在圆周率400；的非常慷慨的价格限制内，这是不是真的很不公平地抱怨这里和那里的一些审美失态！不过，如果消费者有多余的钱可花，也许可以考虑用功能更好的鼠标来代替Pi 400；的鼠标。当你还在使用全尺寸键盘的时候，这可能不是个坏主意。

The integrated keyboard is functional but a solid four inches narrower than a standard keyboard. I'm not generally sensitive to variations in keyboard layout due to a long career involving Other People's Computers in large numbers, but I was plagued with constant mistyping problems the entire time I tested the Pi 400.

集成键盘功能正常，但比标准键盘窄了4英寸。我通常对键盘布局的变化不敏感，因为我的职业生涯很长，涉及到其他人的大量电脑，但在我测试PI400的整个过程中，我一直受到不断的打字错误问题的困扰。

It's also worth noting that, while the Pi 400 supports dual displays, it does so with micro-HDMI ports, not full-sized ones—and it ships with a single cable. You'll need an extra cable if you want to use your Pi 400 with dual displays—and since it ships with a micro-HDMI to HDMI cable, not an adapter, things will get complicated if you want to use it with, e.g., portable LED displays that have off-sized ports themselves.

值得一提的是，虽然PI 400支持双显示器，但它使用的是微型HDMI端口，而不是全尺寸端口-而且它只有一根电缆。如果您要将PI 400与双显示器配合使用，则需要额外的电缆-而且由于它附带的是微型HDMI转HDMI电缆，而不是适配器，因此如果您要将其与本身具有超大小端口的便携式LED显示器一起使用，事情就会变得复杂。

Finally, there's no 3.5mm audio jack on the Pi—if you've got it hooked to a television or a monitor with speakers, it can deliver audio over HDMI; otherwise you'll need a supported USB audio device. I tested with an inexpensive USB gaming headset, which worked fine.

最后，PI上没有3.5 mm音频插孔-如果您已将其连接到电视或带扬声器的显示器上，它可以通过HDMI传输音频；否则，您将需要支持的USB音频设备。我测试了一款便宜的USB游戏耳机，它工作得很好。

I began testing the Pi 400 using its native Raspberry Pi OS Linux distribution—which is basically Debian with XFCE4, and a lot of middleware optimizing it for the Pi. Unfortunately, there's almost nothing in the way of standard benchmarking utilities which run on ARM Linux—all I could find was the Phoronix Test Suite, which would have required more time to run than I had to test the device in total. So for the most part, I'm going to talk about my subjective experience, rather than hard numbers.

我开始使用原生的Raspberry PI OS Linux发行版测试PI 400-它基本上是使用XFCE4的Debian，以及许多针对PI对其进行优化的中间件。不幸的是，在arm Linux上运行的标准基准测试实用程序几乎什么都没有-我能找到的只有Phoronix Test Suite，它运行所需的时间比我测试设备所需的总时间还要多。因此，在很大程度上，我将谈论我的主观体验，而不是确切的数字。

The good news about the Pi 400 is, it does make a credible desktop PC, in the sense that yes, you can totally use it without things breaking. With that said, you're unlikely to forget that you're using a very inexpensive ARM device—much like the Pinebook Pro, the Pi 400 exhibits heavy latency when opening applications that's perhaps possible to live with, but impossible not to notice.

关于PI 400的好消息是，它确实是一台值得信赖的台式PC，从这个意义上说，是的，你完全可以使用它而不会损坏东西。话虽如此，您不太可能忘记，您使用的是一款非常便宜的ARM设备--与Pinebook Pro非常相似，PI 400在打开可能可以接受但不可能不注意到的应用程序时表现出很大的延迟。

Also like the Pinebook Pro, once the applications are actually  open, they generally run smoothly enough—although we did find the Pinebook Pro's hex-core 2.0GHz big / 1.5GHz little CPU noticeably punchier than the Pi 400's straight 1.8GHz quad-core. The biggest problem I had was with high resolution, high frames-per-second YouTube videos.

同样像Pinebook Pro一样，一旦应用程序真正打开，它们通常运行得足够流畅-尽管我们确实发现Pinebook Pro的六核2.0 GHz大/1.5 GHz小CPU明显比PI 400'；的直接1.8 GHz四核更强大。我遇到的最大问题是高分辨率、高每秒帧数的YouTube视频。

I only tested the Pi 400 with a 1080p monitor, so I can't speak to its chops with 4K videos—but it's absolutely not capable of handling the Costa Rica in 4K 60fps HDR video without visible frame drop, even at 720p. The major issue here appears to be the 60fps rate, not the 720p resolution. I also tested Netflix's Our Planet—Forests on YouTube at 1080p, and that video played back flawlessly.

我只用1080p的显示器测试了PI 400，所以我不能用4K视频跟它的排骨说话--但它绝对无法在4K 60fps的HDR视频中处理哥斯达黎加，即使是在720p的情况下也是如此。这里的主要问题似乎是60fps的速率，而不是720p的分辨率。我还在YouTube上测试了Netflix播放的1080p的“我们的星球森林”(Our Planet-Forest)，这段视频播放得无懈可击。

Examining CPU utilization during playback of the 60fps Costa Rica video, we can see the little 1.8GHz Broadcom quad-core CPU struggling—it's at its limits, with CPU utilization for all cores at more than 90 percent. Although the BCM2711 supports hardware offload of video decoding—without which, this video would be playing in seconds per frame, rather than just dropping frames a bit—the hardware offload can only do so much, and the CPU is being asked to take on more than it can handle in software.

检查播放这段60fps哥斯达黎加视频时的CPU使用率，我们可以看到1.8 GHz的Broadcom四核CPU正在苦苦挣扎-它已经达到极限，所有内核的CPU利用率都超过了90%。虽然BCM2711支持视频解码的硬件卸载-如果没有硬件卸载，该视频将在每帧数秒内播放，而不仅仅是丢帧-硬件卸载只能做这么多事情，并且CPU被要求承担超过其软件处理能力的任务。

This effect is even more visible when entering or leaving full screen playback. On a standard desktop PC, that operation takes perhaps 100-150ms. On the Pi 400, it frequently takes as much as three or four full seconds, during which the video itself tends to keep playing, but the surrounding controls and framework only partially render / stop rendering while the shift finishes taking place.

进入或退出全屏播放时，此效果更加明显。在标准的台式PC上，该操作可能需要100-150ms。在PI 400上，通常需要长达3到4整秒的时间，在此期间，视频本身往往会继续播放，但周围的控件和框架只会在移动结束时部分渲染/停止渲染。

Getting audio out of the Pi 400 was a bit of a challenge; it defaulted to attempting to deliver audio over HDMI, and Raspberry Pi OS's audio control dialog isn't the best. Even after changing the output device to USB Audio (my gaming headset), YouTube wasn't producing audio—and there's no "test" button I could find in Pi OS, like the one in Ubuntu's audio control dialog. Closing and re-opening the browser entirely after changing the output device resolved the issue, and audio played from the headset fine afterward.

从PI 400中提取音频有点困难；默认情况下，它会尝试通过HDMI传输音频，而Raspberry Pi OS的音频控制对话框也不是最好的。即使将输出设备改为USB Audio(我的游戏耳机)，YouTube也不会产生音频-而且我在Pi OS中也找不到测试按钮，就像Ubuntu的音频控制对话框中的按钮一样。更换输出设备后完全关闭并重新打开浏览器解决了问题，之后可以从耳机正常播放音频。

All of these quibbles aside, I again have to make note of the sheer inexpensiveness of the Pi 400—it's only $70 for the device itself, or $100 for a kit which includes a mouse, SD card, micro-HDMI to HDMI cable, USB-C power supply, and 247-page full-color guide packed chock full of tips and projects.

撇开所有这些吹毛求疵不谈，我不得不再次指出，PI400-It的价格非常便宜，设备本身的价格只有70美元，或者100美元的套件，其中包括鼠标、SD卡、微型HDMI转HDMI电缆、USB-C电源和247页的全彩色指南，里面装满了提示和项目。(注：PI400-It的价格只有70美元，或者100美元的套件，其中包括鼠标、SD卡、微型HDMI转HDMI电缆、USB-C电源和247页的全彩色指南，里面装满了提示和项目。

At $100 or less for a functional, reliable, well-packaged and integrated desktop computing device, I'm not going to get mad about YouTube looking funny and hanging for a few seconds when it shifts back and forth from full screen. Yes, Walmart's $350 Gateway laptop is considerably more powerful, and includes a screen, battery, and much better keyboard... but that $350 would buy  five Raspberry Pi 400s.

一款实用、可靠、包装良好和集成的桌面计算设备售价不超过100美元，我不会因为YouTube在全屏来回切换时看起来很滑稽，还挂了几秒钟而生气。是的，沃尔玛售价350美元的Gateway笔记本电脑功能要强大得多，而且包括屏幕、电池和更好的键盘……。但这350美元可以买到5辆树莓PI 400。
