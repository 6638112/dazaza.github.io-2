Windows Defender Bug可以用数千个文件填充您的硬盘

2021-05-06 08:17:18

根据用户报告和我们的测试，Windows Defender正在删除Windows 10的系统驱动器上的数千个文件。已发现错误的Windows Defender Update对混乱负责的用户已被广泛报告。

Microsoft Defender预先安装在所有版本/版本的Windows 10上，并通过Windows Update定期更新。在4月的最后一周到最近的Windows Defender之后，用户注意到Windows已开始在系统驱动器中生成数千个文件I.E安装操作系统的位置。

受影响的用户在Microsoft论坛和Reddit上报告了他们的Windows Defender历史文件夹（C：\ ProgramData \ Microsoft \ Windows Defender \ Scans \ History \ Store）悄悄地提起文件。文件本身是微小无害的，但它们可以在某些情况下消耗大量的存储空间。

该错误还击中了Windows Server安装，其中一些人报告其在其组织中生成了1800万个文件。

“到目前为止，我们有三个2016年服务器受到影响。我们的HDD Space警报昨晚开始脱落。一台服务器在商店文件夹中有1800万个文件。另一个人有1300万。拍摄几个小时只是为了发现所有文件，以便可以删除它们。由于分配大小，这些文件在磁盘上占用50-60GB。这是来自微软的一个重大螺钉，“Reddit上的用户指出。

检查Windows Defender是否困扰着该错误的最简单方法是通过File Explorer：

打开文件资源管理器并确保在“视图”选项卡上允许“隐藏项目”。

如果您受到影响，您将看到数百或数千个具有少于一个兆字节大小的文件。 您可以安全地删除文件夹中的文件。 好消息是，问题似乎已通过Windows Defender Engine 1.1.18100.6解决。 要应用修补程序，请打开“开始”菜单，然后单击“设置Cog”图标。 在“设置”应用程序中，导航到更新＆amp; 安全＆gt; Windows更新，然后单击检查更新。 标题为“Microsoft Defender AntiVirus的安全智能更新”的新更新将自动开始下载和安装。 您可以通过在Windows安全性应用程序中打开“关于”页面来验证安全引擎的版本号。
