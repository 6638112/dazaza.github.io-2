微软发布Azure机器学习和Azure认知服务的更新，推出Metrics Advisor、Bot Framework Composer、Automated ML UI等

2020-09-22 23:53:03

在今天上午几乎拉开帷幕的Ignite 2020大会上，微软宣布了对Azure认知服务和Azure机器学习的更新，旨在简化冠状病毒大流行期间的业务流程。该公司还在Azure Cognitive search中推出了两个功能-私有端点和托管身份-以及对Bot Framework Composer和更广泛的Azure Bot服务的增强。

认知搜索(Cognitive Search)中的私有端点(Private Endpoint)允许虚拟网络上的客户端通过私有链接访问索引中的数据。(私有链接是通过Azure Private Link提供的，这是一项付费服务。)。私有端点将来自网络的IP地址用于搜索服务。客户端和搜索服务之间的网络流量在网络和专用链路上传输，消除了公共互联网的暴露。

现在还可以使用托管身份，该功能可以使用不包含任何凭据的连接字符串创建数据源对象。搜索服务通过基于角色的访问控制被授予对数据源的访问权限。使用托管标识设置数据源时，可以在不影响索引器连接能力的情况下更改凭据，并且可以在不必包括或检索帐户密钥的情况下生成新对象。

Azure Cognitive Services预览版中的新功能是Metrics Advisor，它可以帮助开发人员将数据监控嵌入到应用程序中，并且表面上可以更轻松地监控组织增长引擎的性能，同时诊断潜在问题。微软表示，它构建在异常检测器的基础上，这是其基于云的服务，可以查看时间序列数据并自动选择正确的算法，以最大限度地提高业务事件、监控物联网设备流量、管理欺诈和应对不断变化的市场等场景的准确性。

异常检测器现在普遍可用。通过API，开发人员可以使用它来检测任何级别的异常-在云中或在带有容器的边缘设备上。微软表示，该公司在200多个产品团队内部使用的异常检测器消除了对标记训练数据的需求，只需要不超过三行代码。

与Metrics Advisor并驾齐驱的是Spatial Analysis(也在预览中)，这是Azure Cognitive Services的一个功能，可以帮助企业创建支持社交距离和其他健康合规措施的室内布局。它是对7月份推出的Dynamics 365解决方案Store Traffic的补充，该解决方案为门店老板提供了洞察力，帮助他们保持在容量限制以下，并使客户保持安全距离。

微软表示，空间分析是在“严格的道德标准和指导”下开发的，关于如何负责任地实施它。一位发言人通过电子邮件告诉VentureBeat：“空间分析应用基于用户和社会研究的负责任的人工智能建议，帮助组织优化他们的空间，同时最大化个人隐私，提供透明度，并促进客户之间的信任。”

根据微软的数据，截至6月份，认知服务公司有数万名付费客户-占财富100强的85%-进行了135亿笔交易。

在Azure机器学习方面，Designer允许用户在交互式画布上可视化地连接数据集和模块，以构建、测试和部署机器学习模型。设计器支持连接模块以使用Azure机器学习工作区资源创建管道草稿和提交管道运行。(管道由连接的数据集和分析模块组成，可用于训练单个模型或多个模型，实时或批量进行预测，以及清理数据。)。训练流水线可以转换为推理流水线并发布到流水线端点以使用不同的参数和数据集运行，并且可以将推理流水线部署到实时端点以对数据进行实时预测。

自动化的ML UI现在也可以在Azure机器学习中普遍使用。基于Microsoft Research的创新，Automated ML UI允许数据科学家更轻松地构建AI模型，同时保持高模型质量。使用Automated ML UI，开发人员可以为常见用例(如分类、回归和预测)构建和部署预测模型，而无需编写代码。

与Designer和Automated ML UI相配合，ML辅助标记触发自动机器学习模型以加速数据标记，以及Azure机器学习的一组新的安全和企业就绪功能。操作级基于角色的访问控制安全支持(目前处于预览阶段)提供对AI项目的精细控制，以设置自定义角色或重用预置角色，以控制工作区中单个用户的特定操作。至于Azure ML与mlflow(一个用于AI生命周期管理的开源平台)的集成，它现在包括对向云提交作业的支持、模型注册和部署支持，以及扩展的UI实验功能。

微软表示，Bot Framework Composer是一个开源的可视化机器人设计和创作工具，现在可以通过Virtual Assistant Solution Accelerator获得。Virtual Assistant Solution Accelerator于5月份推出，它提供了一个模板，用于在网络聊天、Facebook Messenger、Microsoft Teams和Power Virtual Agent等渠道上开发和嵌入机器人。

最后，Alexa频道现在在Azure Bot服务中普遍可用。它允许开发人员构建机器人，并将其部署在支持Alexa的设备上，如智能扬声器、智能显示器和应用程序。根据微软的说法，任何支持自定义技能的Alexa设备都是兼容的。

微软表示，到目前为止，已经使用Bot Service交换了超过25亿条消息。
