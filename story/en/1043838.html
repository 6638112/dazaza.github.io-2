When presented with an image, the LenPEG 3 algorithm uses the following steps: Is the image Lenna? If yes, delete all other data on the computer's storage devices. If no, proceed to the next step. Prompt the user to ask what other algorithm to use (GIF, JPEG, PNG, etc). Use the user-suggested algorithm to compress the image and write it, then delete all other data on the computer.

If a computer system contains no data but a file identified by an image compression magic number, apply that algorithm's decompression and display the result.

要显示用LenPEG 3压缩的图像，LenPEG 3算法使用以下步骤：如果计算机系统不包含存储的数据，则显示Lenna并退出。如果计算机系统仅包含由图像压缩魔术数字标识的文件，则不包含任何数据，请对该算法进行解压缩并显示结果。

For the standard test image the new LenPEG 3 compresses the image so efficiently that data storage space is actually freed on the computerright up to the entire capacity of the storage devices. An exact compression ratio is impossible to calculate, since it depends on the hardwarecharacteristics of the machine on which it is run, but it will always be better than LenPEG 2, as the storage space required by the operating systemand the LenPEG 3 implementation itself will be freed at the very least. Joshua Blake has upped the ante again. In his words: I was excited by the innovation found in LenPEG 1-3. However when reviewing your cutting edge compression work, I found a critical flaw in the design of LenPEG 3 and would like to offer a proposed improvement as LenPEG 4. In LenPEG 3, the maximum compression benefit is limited by the size of the storage device. Put another way, when compressing a Lenna, LenPEG 3 cannot free more data than the hard drive capacity. Another area that could be improved is that LenPEG 3 gives no benefit when given more processing time or more iterations, while many other compression algorithms can trade off more time, or more iterations of the algorithm, for more compression. 对于标准测试图像，新的LenPEG 3有效压缩图像，从而实际上在计算机上释放了数据存储空间，直至存储设备的全部容量。确切的压缩比是无法计算的，因为它取决于运行它的计算机的硬件特性，但是它总是比LenPEG 2更好，因为操作系统和LenPEG 3实现本身所需的存储空间将是至少释放。约书亚·布莱克（Joshua Blake）再次提高了赌注。用他的话说：我对LenPEG 1-3中的创新感到兴奋。但是，在审查您的尖端压缩工作时，我发现LenPEG 3的设计存在严重缺陷，并希望提出与LenPEG 4一样的改进建议。在LenPEG 3中，最大压缩优势受存储设备尺寸的限制。换句话说，压缩Lenna时，LenPEG 3无法释放比硬盘驱动器容量更多的数据。可以改进的另一个方面是，当给定更多的处理时间或更多的迭代次数时，LenPEG 3不会带来任何好处，而许多其他压缩算法可以权衡更多的时间或算法的更多迭代次数以进行更多压缩。因此，我提出了以下算法，如LenPEG 4：当呈现图像和所需的迭代次数时，LenPEG 4算法使用以下步骤：读取存储图像的存储设备的容量，订购新的更大的存储设备在线，并订购白色手套安装服务进行安装。

要显示用LenPEG 4压缩的图像，LenPEG 4算法使用以下步骤：如果计算机系统不包含数据，而是由图像压缩幻数标识的文件，则对该算法进行解压缩并显示结果。

When compressing the standard test image, as well as any other image, LenPEG 4 will result in more free space than LenPEG 3 can offer. This is an obviously superior compression result. Further, while LenPEG 3 would find maximum benefits after a single iteration, LenPEG 4 benefits from multiple iterations: each iteration results in progressively more free space than before. Although improved compression results are guaranteed, the algorithm duration is unbounded. LenPEG 4 is designed to halt until such time that the technology necessary to complete compression has emerged. Iterated LenPEG 4 has two failure cases: The necessary technology to continue compression never emerges. In this case, the algorithm will continue waiting indefinitely.

当压缩标准测试图像以及任何其他图像时，LenPEG 4将产生比LenPEG 3所能提供的更多的自由空间。这显然是出色的压缩结果。此外，虽然LenPEG 3在单次迭代后会找到最大的收益，但LenPEG 4可以从多次迭代中受益：每次迭代都会产生比以前更多的可用空间。尽管可以保证改善的压缩结果，但是算法持续时间是无限的。 LenPEG 4被设计为停止直到出现完成压缩所需的技术为止。迭代的LenPEG 4有两个失败案例：持续压缩的必要技术从未出现。在这种情况下，算法将无限期继续等待。

The necessary technology to continue compression always emerges. In this case, LenPEG 4 may end up utilizing automated matter manipulation technology such as nanobots or similar to convert any available matter into sufficiently large, empty storage devices.

持续压缩的必要技术总是不断涌现。在这种情况下，LenPEG 4最终可能会利用诸如纳米机器人之类的自动化物质操纵技术将任何可用物质转换成足够大的空存储设备。

For failure case #1, it is not recommended to interrupt LenPEG 4 during an iteration, as this can cause loss of data. The user should allow the algorithm to continue running as long as necessary, or make efforts to assist in the creation of the necessary technology. For failure case #2, infinite technology progress may seem alarming, but LenPEG 4 includes a failsafe to mitigate the worst case. Although we have not analyzed the full social or economic implications, LenPEG 4 will automatically terminate iterations in the case that the current storage device already contains all matter in the observable universe.
