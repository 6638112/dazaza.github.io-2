GO 1.16减少了生成的可执行文件的大小

2020-11-09 01:54:55

Just tried building VictoriaMetrics project with Go tip (aka upcoming Go 1.16) and noticed significant reduction in produced binary size comparing to Go 1.15.4:

我刚刚尝试使用GO提示(也就是即将推出的GO 1.16)构建VictoriaMetrics项目，并注意到与GO 1.15.4相比，生成的二进制大小显著减少：

$ ls -lGg --si bin/total 34M-rwxr-xr-x 1 19M Nov 7 01:19 victoria-metrics-go1.15.4-rwxr-xr-x 1 16M Nov 7 01:21 victoria-metrics-go-tip

$ls-lgg--si bin/总计34M-rwxr-xr-x 1 19M 11月7日01：19 Victoria-Metrics-go1.15.4-rwxr-xr-x 1 16M 11月7 01：21 Victoria-Metrics-Go-Tip。

VictoriaMetrics executable size has been reduced from 19MB to 16MB - almost 16% reduction. This looks awesome!

VictoriaMetrics的可执行文件大小已从19MB减少到16MB，几乎减少了16%。这看起来太棒了！

It would be great to know which particular changes in Go 1.16 are responsible for such great binary size reduction.

如果能知道Go 1.16中的哪些特殊变化导致了如此巨大的二进制大小缩减，那将是一件很棒的事情。
