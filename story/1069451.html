KartaView：又一个街道级图像平台

2021-07-24 21:01:36

我最近开始探索将与 KartaView 的集成添加到 Map the Paths Uploader 的可能性。 KartaView 成立于 2009 年，当时名为 OpenStreetView。 2016 年，TeleNav 接管了 openstreetview.org 域名，并以自己的名义开始了自己的服务。在一位未具名商标持有人（可能是谷歌）的干预下，该服务更名为 OpenStreetCam。 2019 年 12 月 12 日，TeleNav 将 OpenStreetCam 出售给 Grab Holdings（一家位于新加坡的食品配送服务公司），价格未公开。 2020 年 11 月，OpenStreetCam 更名为 KartaView。 KartaView 的重点是基于道路的图像，其中大部分来自行车记录仪镜头。这解释了为什么 Grab 对收购感兴趣，因为此类数据将对其运营有用。上传到 KartaView 的大多数行车记录仪镜头来自人们使用手机在车内拍摄图像。 KartaView 有一个上传排行榜，在撰写本文时，最大的贡献者 JB Brown 使用 Android 设备上传了 653,091.26k（或 KartaView 中的 97,407,587 点）。上传到 KartaView 的任何图像均由所有者根据 CC-By-SA 许可发布，并且任何用户都可以在 Web 应用程序或 API 中下载每个图像。

我真正喜欢的 KartaView 的一项简洁功能是能够通过使用时间轴快速浏览序列中的图像来重新创建旅程（尽管没有自动播放可用）。与 Mapillary 一样，KartaView 与 OpenStreetMap 有很好的集成，以改进地图数据，包括通过他们自己的 OpenStreetCam JOSM 插件。在 Web 应用程序中浏览图像时，您也可以使用 OSM 选项卡跳转到其他编辑工具。也就是说，我无法将其显示在 UI 上，也没有看到通过 KartaView API 公开的检测数据。他们的所有代码库都在 GitHub 上开放，包括核心网站和移动应用程序（Android 和 iOS），并且都在 MIT 许可下提供。在接下来的一周中，我将花几个小时探索 KartaView 代码库和 KartaView API。下周五回来看看，我应该对我的发现进行详细的技术记录。探索来自世界各地的步行、骑自行车、滑雪和使用许多其他冒险形式的交通工具。通过请求新的报道或获取自己的报道来填补空白。
