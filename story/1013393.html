SuSE发布主要Linux更新

2020-07-22 08:03:47

三大企业Linux发行公司之一的SUSE于2020年7月21日发布了其旗舰操作系统SUSE Linux Enterprise(SLE)15 Service Pack 2的下一版本及其最新的基础设施管理程序SUSE Manager 4.1。

SLE 15 SP2可在x86-64、ARM、IBM POWER、IBM Z和LinuxONE硬件架构上使用。这个新的Linux服务器版本基于Linux5.3内核。这个新的内核版本包括上游特性，比如任务调度器中的利用率钳位支持，以及高能效的用户空间等待。

支持从openSUSE Leap迁移到SUSE Linux Enterprise Server(SLES)。有了这个，您可以尝试免费的社区openSUSE Linux发行版，然后，如果您发现它是适合您企业的好选择，请升级到SLES。

扩展包搜索。通过使用新的Zypper，SUSE的命令行包管理器，命令选项--zypper search-Packages--sysadmins现在可以跨所有SUSE存储库进行搜索，即使是未启用的存储库。这使得管理员更容易找到所需的软件包。

SLE软件开发工具包(SDK)现已集成到SLE中。开发包与常规包一起打包。

Python3：SLE 15完全支持Python3开发。SLE暂时仍支持Python2。

存储库镜像工具(RMT)取代了订阅管理工具(SMT)。RMT允许镜像SuSE存储库和自定义存储库。然后，您可以直接向RMT注册系统。在安全性更高的环境中，RMT还可以代理其他RMT服务器。

通过改进的SLE实时修补提高业务连续性。SuSE声称实时打补丁可将系统正常运行时间增加多达12个月。SLE Live Patching现在还可用于IBM Z和LinuxONE大型机架构。

最新的CPU支持。这包括对富士通A64FX和下一代AMD EPYC处理器的支持。

扩展的NVIDIA虚拟机管理程序支持，此支持有助于处理人工智能(AI)、机器学习(ML)和边缘工作负载。在低端，Raspberry PI 4设备支持可减少能量和散热，并改进安装流程。

使用RabbitMQ、Prometheus和Maven为自动化、项目构建和面向消息的中间件提供更好的DevOps支持。

更轻松地将SAP HANA和SAP S/4HANA服务迁移到云和混合实施。集成商可以将SAP S/4HANA迁移数据添加到部署脚本以实现自动部署。SAP Basis管理员还可以通过对服务器、群集和SAP应用程序数据进行新的监控和可视化来主动管理SAP系统。

在此版本中，SUSE还与Microsoft合作开发了一个解决方案，以收集并以图形方式显示特定于服务器和SAP的操作数据。微软杰出工程师约翰·戈斯曼在一份声明中说：

SUSE和微软长期致力于确保SAP系统的顺利和可靠实施。Microsoft Azure已经对SUSE的Pacemaker Monitoring Exporter进行了标准化，并很高兴看到SUSE Linux Enterprise Server for SAP Applications 15SP2支持此功能。

SUSE工程和创新总裁托马斯·迪·贾科莫(Thomas Di Giacomo)补充说，包括SUSE Linux Enterprise在内的SUSE解决方案都是为IT转型而设计的。客户把他们的业务押在他们的IT基础设施上，我们把我们的业务押在他们的成功上。有了SuSE并使用真正的开源软件，他们可以简化现有环境并简化转型，这将使他们在加快业务创新的同时利用现代计算的优势。

对于SUSE Manager 4.1，这是一个改进的开源基础设施管理和自动化解决方案，可在边缘、云和数据中心环境中降低成本、识别风险、增强可用性并降低复杂性。使用SUSE Manager，无论是在私有云、公共云还是混合云中，您都可以确保服务器、虚拟机、容器和群集安全、健康、合规且维护成本低。这在当今尤其重要，这要归功于冠状病毒大流行的IT员工中断。SuSE Manager 4.1还可以与Salt DevOps程序一起使用。

它的垂直市场兄弟SUSE Manager for Retail 4.1专门针对零售业进行了优化和定制。此版本提供了针对小型商店运营的增强功能、增强的离线功能和Wi-Fi映像管理，以及增强的虚拟机管理和监控功能。同时，它可以将零售环境扩展到数万个终端设备，并帮助实现服务点部署的现代化。

把所有这些放在一起，您就会得到一个既适用于小型企业又适用于大型企业的坚实、完整的开源软件堆栈。SLE 15SP2看起来不仅是SuSE客户的发行版，而且是任何使用Linux的公司都应该尝试的版本。
