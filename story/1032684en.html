三一桌面环境R14.0.9发行说明

2020-11-02 05:36:29

The Trinity Desktop Environment is a complete software desktop environment designed for Unix-like operating systems, intended for computer users preferring a traditional desktop model, and is free/libre software.

三一桌面环境是为类Unix操作系统设计的完整软件桌面环境，面向更喜欢传统桌面模式的计算机用户，并且是免费/libre软件。

R14.0.9 is the ninth maintenance release of the R14.0 series, it builts on and improves the previous R14.0.8 version. Maintenance releases are intended to promptly bring bug fixes to users, while preserving overall stability through the avoidance of both major new features and major codebase re-factoring.

R14.0.9是R14.0系列的第九个维护版本，它构建并改进了以前的R14.0.8版本。维护版本旨在迅速为用户带来错误修复，同时通过避免主要新功能和主要代码库重构来保持总体稳定性。

Albeit being a maintenance release, this version brings along a bunch of new applications to expand the offer of the TDE desktop and provide a more modern look.

虽然这是一个维护版本，但这个版本带来了一系列新的应用程序，以扩展TDE桌面的提供，并提供更现代的外观。

When upgrading from previous releases you may run into some of the following use cases. Follow the recommended instructions to minimize potential problems.

从以前的版本升级时，您可能会遇到以下一些用例。按照建议的说明操作，将潜在问题降至最低。

For the time being you can try one of the following workarounds in case you have troubles with PA (especially muting/unmuting).

目前，如果您遇到PA问题(特别是静音/取消静音)，您可以尝试以下解决方法之一。

You can open KMix mixer window -> Settings -> Configure global shortcuts and remap the "toggle mute" keyboard shortcut to some other key combination. Do not use "XF86AudioMute" key, since this is normally used by KMilo and therefore may not work.

您可以打开kMix混音器窗口->；设置->；配置全局快捷键，并将切换为静音的键盘快捷键重新映射为某些其他组合键。请勿使用"；XF86AudioMatt"；键，因为这通常由KMilo使用，因此可能不起作用。

You can create a script that calls "dcop kmix Mixer-1 toggleMasterMute", this will mute/unmute the PA master channel. You can add an input action tied to a keyboard shortcut to call this easy.

您可以创建一个调用"；DCOP kMix Mixer-1 toggleMasterMant"；的脚本，这将使PA主通道静音/取消静音。您可以添加一个绑定到键盘快捷键的输入操作，称这很简单。

KNotes had four global keyboard shortcuts that were bounded to Alt + Shift + C/N/S/H. These shortcuts were overlapping with other key combinations used in some common programs, for example Midnight Commander.

KNotes有四个全局键盘快捷键，分别限制为Alt+Shift+C/N/S/H。这些快捷键与一些常见程序中使用的其他组合键重叠，例如午夜指挥官。

In R14.0.9 those KNotes global keyboard shortcuts have been left unassigned by default. If you wish to keep using the previous setup, you need to right click on KNotes system tray icon, select "Configure shortcuts" and then assign the following key combinations.

在R14.0.9中，默认情况下未分配这些KNotes全局键盘快捷键。如果您希望继续使用以前的设置，则需要右键单击KNotes系统任务栏图标，选择配置快捷键，然后分配以下组合键。

Starting from this TDE release, apt sources for i386 architecture are no longer provided for Ubuntu 20.04 onwards.

从这个TDE版本开始，从Ubuntu20.04开始，不再为Ubuntu20.04提供i386架构的APT源代码。

If you have a multi-arch system that includes i386 as well, you need to either remove i386 from the list of known architectures or remove that architecture from your apt sources.

如果您的多拱门系统也包含i386，则需要从已知体系结构列表中删除i386，或者从APT源代码中删除该体系结构。

For example, if you have both i386 and amd64, you need to add [arch=amd64] in your apt sources.list file to make sure i386 is not used.

例如，如果您同时拥有i386和AMD64，则需要在apt Soures.list文件中添加[ARCH=AMD64]以确保不使用i386。

Installation instructions and binary packages are immediately available for Debian, Devuan, Raspbian and Ubuntu. Packages for RedHat/CentOs, Fedora, Mageia, OpenSUSE, and PCLinuxOS are being built by their respective maintainers and may be already available or will be soon.

Debian、Devuan、Raspbian和Ubuntu的安装说明和二进制软件包立即可用。RedHat/CentOS、Fedora、Mageia、openSUSE和PCLinuxOS的软件包正在由各自的维护者构建，可能已经或即将推出。

Live CDs with TDE R14.0.9 preinstalled are or will be available soon on the LiveCDs page.

预装了TDE R14.0.9的Live CD已经或即将在LiveCDs页面上提供。

TDE continues to offer support for FreeBSD, although still partial given some of the new issues that have arisen with the latest versions of FreeBSD. The TDE team will try their best to improve support for FreeBSD but progress will probably be slow. Additional developers or contributors willing to help porting the remaining features and applications are always welcome.

TDE继续提供对FreeBSD的支持，尽管考虑到最新版本FreeBSD出现的一些新问题，它仍然是部分支持。TDE团队将尽最大努力改善对FreeBSD的支持，但进展可能会很慢。愿意帮助移植剩余功能和应用程序的其他开发人员或贡献者总是受欢迎的。
