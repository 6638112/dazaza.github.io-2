开放性问题：我们如何修复文件？

2021-08-06 13:21:36

我最近正在帮助我妈妈使用 Microsoft Sharepoint 共享文件。该文件是包含一些 Word 文档的 zip 存档。我们上传了文件并向收件人发送了一个共享链接。他们单击了链接，但无法使用这些文件。事实证明，该链接不是下载文件，而是打开一个在线文件浏览器，该浏览器导航到 zip 文件中。这引起了混乱，因为用户不明白文件的在线视图与在您的计算机上保存文件不同。这在在线文件领域很常见。所以我开始思考：我们如何才能构建出不会持续让人们感到困惑的软件？上下文很重要。如果您在计算机上打开一个文件夹，它与在 Web 浏览器界面中打开它不同，也与在手机上打开它不同。在计算机中，很大程度上取决于上下文。文件就是一个很好的例子。它们是现代计算的基石，人们在许多不同的环境中处理文件。文件浏览器视图是我们行业的标志。任何使用过带有 GUI（图形用户界面）的计算机的人都知道文件浏览器是什么样的。它看起来与经典 Windows 中的相同，中间是文件列表，左侧通常是一些常见文件夹。在任何你找到文件的地方，你最终都会遇到一个看起来很熟悉的文件浏览器。虽然它们看起来相同，但它们的行为并不相同。某些文件浏览器可以创建、编辑和保存文件。有些只能显示它们。有些只能处理某些类型的文件。有些甚至不会显示正确的列表。这通常是平台限制。用文件做某事。用户需要将文件从一个文件浏览器移动到另一个文件浏览器。对于用户来说，这是令人困惑的。我们有这么多不同的浏览器是一件坏事吗？有没有办法简化和精简这个支离破碎的系统？我认为查看这些问题很重要，因为每一个新的文件浏览器和每一个新的上下文都会让人们更难知道它应该如何工作。它使我们的用户孤立，并将他们保留在一个较小的生态系统中。写于 05/08/2021
