数字媒体新闻发布者讨论了他离开Substack的产品和哲学原因，包括对电子邮件列表的控制不足

2020-12-17 03:48:26

大约一年前，我写了一篇关于我准备离开Substack的文章。我度过了假期，在WordPress上重建了网站。我在重新启动网站的过程中走得很远，然后在Substack与团队聊天并决定留下。

一年后，我又回到了这里。这次，我与聪明的人合作，帮助我从构想变为执行。开关声名狼藉，一周后，我们来了。出于所有目的和目的，迁移似乎进展顺利。人们正在接收他们的时事通讯，对于想要存档的人来说，他们的登录情况也很好。 （如果您不在，请告诉我）。

当其他所有人都涌向该平台，并且正经历着最好的几个月时，我为什么决定离开呢？首先，让我解释一下为什么Substack实际上是一个非常好的平台。

当我第一次创建A Media Operator时，这是开始编写，发布和发布新闻简报的最简单方法，而这全是在我从事全职工作的时候。是的，我本可以设置自己的WordPress，然后发布并立即构建新闻通讯。我知道这样做的人。这很麻烦，需要时间，而且坦率地说，我想要更简单的东西。

在2019年8月中旬，首次启动A Media Operator时，这是一个非常简单的主张：当我发表文章时注册并接收更新。我能够非常轻松地开始编写，发布和发送电子邮件。我对此感到高兴，说实话，我认为如果不是那样的话，今天就不会有媒体运营商。

如果您要建立新闻简报/博客，目的是通过订阅获利，那么这可能是正确的方法。现在，您可以拥有自己的域名了，如果您只是建立基本的新闻通讯/博客，那实际上还不是那么糟糕。

不过，媒体运营商已经成立一年半了，我开始将其视为不仅仅是新闻通讯。从长远来看，这会成为一家拥有多种收入来源的媒体公司吗？如果是这样，我是否需要能够支持这种发展的技术栈？

网站重新发布可能会令人讨厌。您必须清楚地知道需要什么与想要什么并确定优先级。尤其是当您进行精益操作时，必须非常集中精力。如果您不这样做，则将花费数年时间尝试构建某些东西，但仍然会陷入困境。

我坐下来考虑了媒体运营商可以进入的各种商机：

超高级社区：也许这只是一个高级主管团队。在从免费到付费的渠道中，这将低于付费。

数据与研究：我应该开始建立有关公司筹集了多少资金，其收入，团队规模，盈利能力，业务模型等的数据库吗？

咨询：我已经开始这样做，所以我的目标是明确提供它。

高管对接会：我收到了很多人的提问，他们问：“您认识适合这个职位的人吗？”招聘人员赚钱。

媒体工作室：整个夏天，我开始考虑媒体工作室的外观。我可以通过集中服务孵化并创立新的媒体公司吗？

活动：也许我为数百人发起了一个活动，获得了一些赞助商，并在Covid之后召集了人们。

在寻找各种机会时，我意识到了一些非常重要的事情……我需要完全控制我的电子邮件列表。

值得注意的是，使用Substack，您实际上不需要。从表面上看，您确实这样做了。您可以建立清单，离开时可以随身携带，也可以随时通过电子邮件发送。但是，这与完全控制我的电子邮件列表不同。

此列表中有许多来自各种出版物的高级管理人员媒体运营商。名单上还包括记者，初级员工，个人创作者和其他各种各样的人。但是，当我准备发送时事通讯时，每个人都是平等的。

进行细分时，我无法控制列表。假设我建立了一个超高级社区。我将如何营销？我显然不想给所有人发送电子邮件；甚至不是我的高级会员中的每个人。对他们来说可能不合适。我需要能够创建细分，而Substack不允许这样做。它是为新闻简报/博客而不是市场营销活动构建的。

我非常感兴趣的另一件事是创建常绿内容以在Google中排名。我收到了一系列有关什么是ESP，DMP，CDP，首字母缩写词，人们应该考虑使用的东西等问题。我曾经想到的一个想法是，我可以创建常绿的内容，以挖掘所有这些工具，为搜索进行优化，然后从那里将用户转换为我的免费新闻通讯。

子堆栈不支持该功能。每条内容都只是一篇博客文章，因此，我无法在网站上创建资源子目录。除此之外，我没有简单的方法可以对内容进行任何优化。这是基本的受众开发活动，我什至做不到。

当我们谈论需要在Substack上发现更多内容的需求时，这就是我们在谈论的东西。让我优化我的内容。 Google为发布商带来了可笑的流量，但是却没有优先考虑的地方。

我想启动的其他页面……联系页面，咨询页面，仅播客页面。我希望能够创建这些特定于特定业务需求的定制页面。

关于受众群体发展，我已经考虑过创建推荐系统。我目前的老板为此创造了惊人的业务。我为什么不应该在这里呢？问题在于，由于Substack是一个封闭的花园，因此它没有可以与任何其他平台连接以支持奖励措施的API。

对我来说越来越重要的另一件事是需要控制组订阅。这是一个动手过程。一家公司会来找我，说他们想让部分或全部员工成为会员。通常，此人已经是付费用户。我要么必须：

退还该人的付费订阅费用，然后告诉他们第二次注册，但这次除外，请选择一个团体计划。

请访问Substack的客户支持，要求他们处理，等待几周，然后仍然必须向用户说明该怎么做。

也许我听起来有些恼火，但由于我对此过程感到尴尬，所以我暂停了$ 2,000的团体订阅。摩擦太大了，所以，我只是等待。我告诉他：“我正在移动系统，然后我们会做。”

当我继续思考要构建的内容与Substack支持的内容时，我意识到自己陷入了困境。据我所知，我将永远无法建立一个超高级社区，封闭式数据库，创建分类的常绿内容，在站点上拥有一个事件页面，然后列表就不存在了。

我已经到了尾巴摇狗的地步……工具决定了我的生意。

做出决定的不只是产品需求。我还需要考虑媒体运营商代表什么，以及相反，Substack的理念。

我写过的第一批文章之一就是关于多元化的重要性。如果一种收入来源枯竭，至少您可以获得另一种。

对于整个“创造者经济”，我感觉甚至更多。每个人都应该获得有偿新闻通讯的想法很奇怪。某些内容应直接付款，而其他内容应用作间接获利的工具。我知道创作者可以免费写作，而仅通过咨询就可以获得可靠的六位数的业务。

更明确地说，我想其中许多创作者将意识到在广告中可以赚钱。只需看看Packy在“无聊”中的构想即可。他的名单已经增加到25,000，并且通过销售非常好的广告获得了可观的收入。

我已经看到创作者的发展。当他们第一次进入Substack时，他们对广告的糟糕程度表示敬意，并认为这会扭曲他们为读者制作的产品。快进了六个月，突然之间，他们意识到广告实际上是一个完全可行的选择，尤其是如果您以干净的方式进行广告。

另一方面，子堆栈从根本上反对广告。这不是什么新鲜事。但这让我感到奇怪……Substack是否反对广告，我告诉我认识的所有人（酒吧和创作者）他们应该考虑广告策略，我是否与Substack在哲学上没有冲突？如果是，为什么要在该平台上发布？

我并不是说Substack需要构建类似广告服务器的工具或类似工具。但是，一种简单的方法是计算链接的点击次数而不是点击百分比呢？那太好了。

虽然不是真正的哲学，但我认为Substack需要解决一个更为关键的问题……谁是它的客户？

当您来到媒体运营商时，您就是我的客户。然后，我成为Pico的客户，Pico是我为AMO业务提供动力的CRM /注册系统。但是您实际上不是Pico的客户。我付给Pico，您付给我，这种关系非常清楚。我不能对Substack说同样的话。

您可以从他们的产品开发中了解很多有关企业客户的信息。随着新功能的推出，他们将从中受益？对于Substack，我不会感觉到它知道客户是谁，因此，其产品开发似乎有些偶然。

以最近发布的阅读器应用为例。在Decoder的一集中，首席执行官Chris Best说：

我们想到的很多事情是读者告诉我们的：“嘿，我现在已经订阅了六个不同的子堆栈，我想全部阅读它们，很糟糕的是它们与我的所有其他物品都在我的电子邮件收件箱中。

“如果能有一个地方可以看到我拥有的所有这些直接联系，并且在很多情况下我都愿意为此付费，那不是很好吗？”这是我们非常感兴趣的事情。

坦率地说，他们正在构建Google阅读器。这很酷。我们许多人希望Google阅读器仍然存在。除此以外，还有一个问题：此工具对创作者有何帮助？我不知道怎么做。它服务于其他客户，而不是创建新的媒体公司可能需要发展的工具。

考虑一下Shopify。他们从“武装叛军”的立场行动。他们的顾客是商店的老板，他们花费所有的时间竭尽所能来帮助该顾客赚更多的钱。

Substack构建了一个很棒的工具，用于发送新闻稿，发布博客甚至主持播客。但是后来它停了下来。它从未涉足构建用于受众开发的工具。它把重点转移到为读者服务上。很好，但是对于您的主要客户（创建者）而言，迭代很慢，但这并不是一件好事。

首先，媒体运营商是一家数字媒体公司。但是，当记者们写这篇文章时，他们会将其称为“子堆栈”，例如博客或网站。我也不相信工具的品牌与我的品牌相同。听起来有些微不足道，但多个品牌却令人困惑。

其次，因为他们的品牌是如此重要和中心，所以我必须非常警惕保护我的利益。我最近偶然发现了Substack上的一本铁杆右翼的新纳粹出版物。我不知道它是否仍然存在，但是如果人们打算将我的网站称为“子堆栈”，那么该平台上将会出现新纳粹分子，这可能会损害我的品牌。

当我上周宣布更换产品时，我分解了所使用的工具：

这些是为我的媒体公司提供动力的五种工具。但是，我将第一个告诉您我不是开发人员。早在八月，我在推特上说我正在寻找人，很幸运能邀请A Media Operator的成员伸出援手。

本·梅（Ben May）是The Code Company的董事总经理，该公司专门致力于基于WordPress的数字出版商。他们的大多数客户是澳大利亚的大型媒体公司。但是我与他合作的原因是他了解我正在尝试建立的东西。因为他读过《媒体运营商》，所以我不必花时间分解我的工作。

与Ben的过程很简单。我向他介绍了我长期考虑的功能。我们讨论了一些我特别喜欢的设计风格，然后他离开了，让他的团队模拟媒体运营商的模样。然后，他的团队构建了该网站，运行了一个基本脚本，以将我的内容发布到Substack上到WordPress（这并不像您想象的那么容易），并且在您不知道它的情况下，网站就完成了。

我对Ben说的一件事是，我希望该业务的运营与一键式发布/发送时事通讯在Substack上的运行相同。他想出了如何将WordPress连接到Mailchimp，以便当我在WordPress中点击发布时，它也会发送新闻通讯。

与Ben进行的对话中很大一部分是我们应该如何管理订阅。我们研究了很多系统。在反复讨论今天的业务和发展方向之后，我们同意Pico是适合我的工具。

从本质上讲，Pico确实做到了两件事：它存储用户数据，并允许您创建各种订阅包。我可以根据用户来自何处创建规则，可以管理组订阅，实际上可以专注于建立业务。

这并不意味着Pico是完美的工具。我不喜欢这些表格。我想要对样式和用户流程进行更多控制。我不是JS支持注册系统的最大支持者，因为它会降低运行速度。但是，我已经看到了即将推出的V2的外观。我将获得更多所需的自定义内容。 CRM将变得更加强大。我已经与团队就路线图进行了交谈，并且我相信当我准备开始向A Media Operator添加更多复杂功能时，Pico将可以为我提供支持。

Pico知道其客户是谁。 读者可能会通过Pico的注册系统，但最终，我是Pico的付费用户。 这意味着它将继续构建使我成长的工具。 看...子堆栈很棒。 它做了我当时需要做的事情。 但是我对这家公司的野心远不止于此。 因此，我与The Code Co的Ben和Pico的Jason进行了合作。 我为未来感到兴奋。 如果您想加入A Media Operator的下一阶段并获得更多，请考虑升级为高级会员。 您可以在这里进行。
