Mac OS X Snow Leopard的回顾展

2021-02-24 21:39:53

我最近的一篇文章，重塑MAC经验，从Twitter和WordPress Analytics的回复中获得了很多关注判断 - 显然是其他地方，它到达黑客新闻和Reddit。与我的四部分系列Mac OS Catalina不同：更多的麻烦比价值更麻烦，但是，它根本没有吸引任何仇恨邮件。我收到的纯粹大部分反馈非常积极，很多人都同意我和我的观察。一些 - 一些挑衅性的，一些真正好奇 - 问我沿着那条线，好吧，如果你不喜欢当前的大血管UI和MAC体验，那么你喜欢的Mac OS UI和体验的例子是什么？

这是一个不仅仅是公平的问题，这件作品用作答案。当我写回来那些问我的人时，我回答了Mac OS x 10.6雪豹。这是一种基于良好的回复的肠道回复，这些回复非常广泛地使用该Mac OS版本。

当我在2009年7月购买了我的15英寸MacBook Pro时，它带来了Mac OS x 10.5.7（豹），但我立即升级到雪豹，后一个月左右。如你所知（如果你不这样做，这里是一种进修），与Mac OS X 10.4 Tiger一起，Snow Leopard是Mac OS版本之一，寿命最长，差不多两年，从2009年8月到2011年7月，何时最终的10.6.8 v1.1轻微发布出来了。在我的2009年MacBook Pro上，我继续使用它直到2012年中期，因为Mac OS X 10.7狮子（2011年7月发布）起初并没有完全说服我，所以我等待至少在升级前至少推出10.7.3版。

所以，我在2009年MacBook Pro上使用了Snow Leopard约三年，然后再次在2010年Mac Mini上的朋友让我保持维护，作为一种异地备份。将Mac Mini保存在Mac OS X 10.6.8上，整整四年来在我的监护下（2011-2015），在此期间只关掉两次，也可以总共重新启动四到五次。它享受了一个疯狂的正常运行时间，这是雪豹的稳定性。

但回到我的“GUT-回复”，我想确定我对雪豹的美好回忆不仅仅是怀旧。虽然当我说雪豹是最稳定的Mac OS版本时，我很有信心，我想确保其用户界面真的是我记住的良好用户界面和体验。因此，在使用Mac OS High Sierra上创建虚拟机的令人沮丧的尝试后，我决定在USB闪存驱动器上安装雪豹，并启动我的2009 MacBook Pro（是的，它仍然存在＆amp;踢）在雪豹中，从那个闪存驱动器。

由于MacBook Pro不再具有光学驱动器，因此我必须从我的原始雪豹DVD安装程序创建一个可启动的USB闪存盘。最快的方法是使用磁盘实用程序 - 而不是旧版本的磁盘实用程序，从此应用程序真的是一个实用程序，并且您可以使用还原功能可靠地克隆到（在这种情况下）外部的可启动DVD体积。

从可启动的USB闪存驱动器到另一个USB闪存驱动器，安装相对较快，大约20-25分钟。虽然我对速度首选外部SSD，但我必须说使用雪豹从闪光灯驱动器仍然是微风。系统响应，我没有注意到任何特定的滞后。

我一直在谈论“透明度”，而实际上是“透明度”，至少在Snow Leopard中。

当我在Twitter上分享此内容时，马里奥·古兹曼（MarioGuzmán）观察到，事物按颜色很好地划分了。您可以清楚地分辨出窗口的每个部分（甚至是该死的滚动条）……不仅仅是一个带有灰色符号的白色斑点。

就是这个窗口具有明显可区分的区域：标题栏（在窗口的左上角带有信号灯控件，而右上方的侧边栏+工具栏显示/隐藏切换按钮），工具栏，侧边栏（带有彩色图标可以帮助您快速，轻松地找到项目），路径栏，状态栏，最后是始终可见的滚动栏。

在可用性方面，永久性的向上/向下箭头和滚动条是正确的选择，并且这种用户友好的设计。 “浅蓝色”栏的长度立即使您了解刚打开的文件夹的填充量。此外，如果您需要快速向下滚动，只需用鼠标指针抓住滚动条并滚动即可。

在更高的Mac OS版本中，滚动条默认设置为仅基于鼠标/触控板的移动而出现，这是很遗憾的。许多用户可能没有意识到他们可以使滚动条永久显示，因此他们不必为滚动条出现而将鼠标悬停在时间上，而是希望当他们想快速向下滚动时可以设法抓住它。一长串元素。

我再一次想起WWDC 2020的Alan Dye（苹果公司人机界面副总裁）那句臭名昭著的话，谈到Big Sur的UI重新设计：‌我们降低了视觉复杂性，使用户可以继续关注用户的内容。按钮和控件会在您需要时显示，而在您不需要时会后退。我仍然相信这通常不是一个好方法，尤其是对于滚动条之类的基本元素，默认情况下应始终可见，因为它们是UI元素，其有用性不仅限于您使用它们或与之交互时-即使没有严格要求，它们也会发出信号。对于滚动条，它是文件夹中包含多少元素，项目列表有多长时间以及更重要的是滚动时当前位置的直观估计。

返回Finder窗口，这里是“ Apple对细节的关注”细节：注意窗口左下方的图标吗？这是一个细微的视觉提示，它告诉您Finder图标（项目）是已排序，未排序还是仅捕捉到网格。从只读卷中打开窗口时，此处将显示划线铅笔的图标，这意味着您无法修改随附的项目或写入该卷。

尽管我认为该UI细节不是至关重要的，但拥有它当然很好，并举例说明了使Mac界面更出色的那些小事。如上所述，它反映了对细节和整体思想的一定关注，在更高版本的Mac OS中逐渐消失了。

为什么应用程序窗口的上部没有固定的颜色或外观？在没有侧边栏的“天”视图中，窗口顶部为浅灰色，并且与其余元素分开，就像Big Sur的Finder窗口中的标题栏/工具栏一样。但是在没有侧边栏的“周”视图中，窗口顶部变为白色，并与应用程序窗口的其余部分融合。当然，当显示侧栏时，窗口的顶部会再次更改，因为在Big Sur中，侧栏覆盖了窗口的整个左侧，包括带有信号灯控件的左上区域。

从用户界面的角度来看，我一直很喜欢旧的“系统偏好设置”的主窗口如何将偏好窗格整齐地分类，并且尽管Big Sur的“显示全部”按钮不是特别隐秘，但我总是更喜欢使用文本的清晰度在按钮上，尤其是在有足够空间时。我对当前将应用程序界面中的所有内容都图标化的趋势感到恼火，尤其是当替换文本标签的图标确实不如设计师认为的那样自解释时。

我也喜欢在Snow Leopard中，“ Show All”按钮显然是一个按钮，而在Big Sur中，就像在其UI的许多其他地方一样，您只有在将鼠标悬停时才意识到它是一个按钮（而不仅仅是装饰性图标）超过它。

这可能更多是个人喜好（原谅双关语），但我发现在雪豹号下的图标设计比在大苏尔号下更一致。如果您查看Big Sur中各个首选项窗格的图标，它们似乎是由不同的人绘制的，他们在设计阶段从未在他们之间进行过交流。他们似乎并没有遵循您在系统其他地方找到的统一的美学或图标语言。相反，Snow Leopard中的图标类似于您在其他应用程序的工具栏或“偏好设置”窗口中找到的图标。
