The first trick here is that function parameters evaluation order is unspecified,meaning that new Widget might be called, then priority(), then the valuereturned by new Widget is passed to std::shared_ptr<Widget>(…).The second one is there since this is C++, priority() might raise an exception,meaning that new Widget will be called, but never passed to std::shared_ptr, and thus never deleted!
