You can use as much or as little of tmux as you want. You can set up tmux so that all it can do is select and copy text. You don’t have to have splits; you can keep using your native tabs. You can have the good stuff without any of the punctuation soup.

您可以根据需要使用尽可能多的TMUX。您可以设置TMUX，以便它可以做的一切都是选择并复制文本。你不必有分裂;您可以继续使用您的本地标签。你可以拥有没有任何标点符号的好东西。

Here’s an example of a starter ~/.tmux.conf that lets you select text, and pretty much nothing else:

这是一个Starter〜/ .tmux.conf的示例，可以允许您选择文本，而且几乎没有其他东西：

$ cat ~/.tmux.confset -g escape-time 1set -g mouse onset -g status offset -g prefix S-C-M-wset -g mode-keys vibind -n M-Space copy-mode

$ cat〜/ .tmux.confset -g转义时间1set -g鼠标onset -g状态offset -g前缀s-c-m-wset -g mode-keys vibind-n m-space复制模式

q will exit “copy mode.” To start a selection, use Space instead of v: v toggles block and character mode, confusingly. You can also set -g mode-keys emacs, if that’s your thing. Run tmux list-keys | grep -- '-T copy-mode-vi' or tmux list-keys | grep -- '-T copy-mode' (if you’re using emacs) to see all the keybindings.

q将退出“复制模式”。要启动选择，请使用空间而不是V：V令人困惑地切换块和字符模式。您还可以设置-g mode-keys emacs，如果这是您的事情。运行tmux列表键| grep - ＆＃39; -t copy-mode-vi＆＃39;或tmux列表键| grep - ＆＃39; -t复制模式＆＃39; （如果您正在使用Emacs）以查看所有键键。

You can, of course, rebind anything you want. tmux is very easy to customize; it’s easy to pick and choose exactly what you want. I wrote a whole blog post about setting up tmux à la carte. It will walk you through the basics of setting up a tmux config, demystify tmux’s if statement, and explain how to make tmux’s copy commands integrate with the system clipboard (which might just work out of the box, depending on your terminal emulator, but it might require another line or two in your config).

当然，您可以重新撤销您想要的任何内容。 TMUX非常容易定制;很容易挑选，并确切地选择你想要的东西。我写了一个关于设置Tmux单点菜肴的整个博客文章。它将通过设置Tmux配置的基础知识，揭示TMUX IF语句，并解释如何使TMUX的副本命令与系统剪贴板（这可能只是在框中工作，而根据您的终端仿真器，但它可能需要在CONFIG中提供另外两条线）。
