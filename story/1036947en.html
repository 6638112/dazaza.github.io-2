Htmx 1.0.0 Release
Htmx 1.0.0版本

2020-11-28 07:57:59

htmx is now mature enough that I can recommend it as a general replacement for intercooler.jsprojects. I don't think there is a strong reason to port an existing intercooler project tohtmx. I have several large intercooler apps and will not be moving them over any time soon. I can, however, recommend using htmx over intercooler for new projects.

htmx现在已经足够成熟，我可以推荐它作为intercooler.jsprojects的常规替代品。我认为没有充分的理由将现有的中冷器项目移植到htmx。我有几个大型中冷器应用程序，并且不会在短期内转移它们。但是，对于新项目，我可以建议在中冷器上使用htmx。

htmx is a different sort of javascript library. It is an HTML & hypertext-oriented reply to the current dominance of javascript-based SPA libraries. It is a response to Tom MacWright's question: "If not SPAs, What?".

htmx是另一种JavaScript库。它是对HTML和超文本的一种回应，以应对基于javascript的SPA库的当前主导地位。这是对汤姆·麦克赖特（Tom MacWright）问题的回答：“如果不是SPA，那是什么？”。

HTML-oriented web development was abandoned not because hypertext was a bad idea, but rather because HTML didn't have sufficient expressive power. htmx aims to fix that & allows you to implement many common modern web UI patterns using the original hypertext model of the web.

放弃基于HTML的Web开发并不是因为超文本是一个坏主意，而是因为HTML没有足够的表达能力。 htmx旨在修复该问题，并允许您使用Web的原始超文本模型来实现许多常见的现代Web UI模式。

In april of this year I began work on a jQuery-indepenent & improved version of intercoolerjs, renamedto htmx. I chose to rename the library because, in working on intercooler, I had come to appreciate that intercooler & htmx were completing HTML as a hypertext rather than just some funky, idiosyncratic javascript libraries.

在今年4月，我开始研究jQuery独立和改进的intercoolerjs版本，并将其重命名为htmx。我选择重命名该库，是因为在使用intercooler时，我逐渐意识到intercooler和htmx正在将HTML作为超文本完成，而不仅仅是一些时髦的，特殊的javascript库。

In May htmx reached 0.0.1. Soon thereafter I had the good fortune of being contacted by Ben Crokerwho was interested in htmx as a base for his new reactive library, Sprig. Ben was willing to be an early adopter of htmx and pushed the library alongmuch faster than it would have gone otherwise.

5月htmx达到0.0.1。此后不久，我很幸运地被Ben Crokerwho联系了，他对htmx感兴趣，他将htmx作为他新的反应式库Sprig的基础。 Ben愿意成为htmx的较早采用者，并且比其他方式更快地推动了图书馆的发展。

I have been very lucky to the have help and feedback from many contributors in Github and on Discord. I'd like to thank, in particular, Ben Pate, Robert Schroll & Alejandro Schmeichler for contributing code as well as new ideas and discussions.

我很幸运能在Github和Discord上获得许多贡献者的帮助和反馈。我要特别感谢Ben Pate，Robert Schroll和Alejandro Schmeichler贡献了代码，并提出了新的想法和讨论。

I would like to thank Devmode.fm for having me on to talk about htmx and for cleaning up all my "uhhs" and "umms".

我要感谢Devmode.fm让我继续谈论htmx并清理我的所有“ uhhs”和“ umms”。

Finally, I would like to thank Justin Sampson, who took a lot of time to explain REST & HATEOAS to me and how intercooler (and now htmx) fit into that model for web development.

最后，我要感谢贾斯汀·桑普森（Justin Sampson），他花了很多时间向我解释REST和HATEOAS，以及中间冷却器（现在是htmx）如何适合该Web开发模型。
