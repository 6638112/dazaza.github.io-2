Android版Chrome将显示基于Web Vitals的“快速页面”标签

2020-08-18 03:04:52

谷歌今天宣布，Chrome for Android的上下文菜单将为被认为具有良好性能的网页显示“快速页面”标签。标签将使用谷歌的Web Vitals来确定，该公司在5月份宣布了一项倡议，旨在为网站开发人员和网站所有者提供一套统一的衡量标准，以便在考虑用户体验和性能的情况下建立网站。核心网络活力，谷歌试图阐明它认为对所有网络体验至关重要的指标，衡量网页的响应性和视觉稳定性。

同样在5月份，谷歌分享了一项计划，将网页体验作为谷歌整体搜索和移动热门新闻专题的排名因素。谷歌承诺今年不会改变排名，并在改变排名前六个月发出通知，但由于冠状病毒的普遍不可预测性，谷歌不愿承诺具体的时间安排。不过，如果你改进了你的网页指标，谷歌很乐意在Chrome中给你的链接贴上“快速”的标签。换句话说，谷歌不想在大流行期间使用棍子，但胡萝卜完全可以。

谷歌诞生于网络，其收入与网络直接相关。改善网络的用户体验符合该公司的既得利益。考虑到谷歌的影响力，包括超过10亿Chrome用户和超过25亿台每月活跃的Android设备，更不用说谷歌搜索了，任何拥有网站的人都需要跟踪谷歌对网络的优先事项。如果搜索排名因素还不足以引起网站开发人员和网站所有者的注意，Chrome的变化应该会让他们重新考虑。

从Chrome85for Android(目前处于测试阶段)开始，Google将通过链接上下文菜单标记快速链接。Chrome for Android的改变是让Chrome“突出网络上的高质量用户体验”这一更大计划的一部分。事实上，谷歌表示，它“可能还会在Chrome用户界面的其他部分尝试标签。”

标签基于核心Web Vitals指标的信号，这些指标衡量Web可用性的维度，如加载时间、响应能力和内容加载时的稳定性。谷歌表示，历史上达到或超过Core Web Vitals所有指标阈值的页面链接将被贴上新的“快速页面”标签。当您在导航到页面之前长按链接时，您将看到标签。

网站所有者为改进这些方面所做的改变致力于让所有网络浏览器的用户对网络更感兴趣。投资于这些以用户为中心的关键指标有助于推动用户可用性的提高，并帮助企业看到更多的参与度。

谷歌表示，如果一个或多个类似的URL“在历史上对其他用户来说一直都很快”，那么“快速页面”标签可能会出现在链接上。换句话说，链接对您来说不一定很快。当谷歌没有足够的URL数据来评估速度时，标签将基于来自具有类似结构的网站URL的聚合历史数据。例如，当URL是新的或不太受欢迎时，可能会发生这种情况。

谷歌计划每年更新Core Web Vitals。该公司在5月份表示，这些变化将向下渗透到谷歌搜索排名。今天，该公司表示，随着Core Web Vitals的发展，Chrome标签的变化也将保持一致。这样，Chrome应该根据“最能代表用户整体体验的指标”来标记页面。

为了针对核心Web Vitals进行优化，网站所有者可能不得不投资于提高页面质量。谷歌已经更新了其Web Vitals工具，以便在LighTower、DevTools、PageSpeed Insights和搜索控制台上显示信息和推荐。

虽然标签功能目前正在向Chrome85Beta推出，但您现在可以通过转到Chrome：//FLAGS并启用“上下文菜单性能信息和远程提示获取”来对其进行测试。然后打开一个网页，长按任意链接。

谷歌使用标签来鼓励它想要的行为已经有很长的历史了。2014年11月，谷歌开始给网站贴上“移动友好型”的标签，以表示为手机优化的页面。该公司随后尝试使用标签作为排名因素，最终在2015年4月推动了这些变化，并在2016年5月提高了影响。谷歌在2016年8月去掉了这个标签，指出大多数页面已经变得“对移动设备友好”。2018年，谷歌搜索(Google Search)开始将速度更快的移动页面排名更高。

该公司对Chrome和HTTPS进行了类似的测试。2017年1月发布的Chrome 56开始将收集密码或信用卡的HTTP页面标记为“不安全”。2017年10月发布的Chrome 62开始将输入数据的HTTP网站和所有以隐姓埋名模式查看的HTTP网站标记为“不安全”。2018年7月发布的Chrome 68将所有HTTP网站标记为“不安全”。最后，2018年9月发布的Chrome 69从HTTPS网站中删除了“安全”的措辞。你就明白了。如今，Chrome用户97%的浏览时间都花在HTTPS上。
