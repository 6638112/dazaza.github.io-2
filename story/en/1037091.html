Microsoft is working on an Android subsystem for Windows 10
微软正在为Windows 10开发一个Android子系统

2020-11-29 10:41:55

Microsoft is reportedly creating a subsystem, similar to the Windows Subsystem for Linux, that allows Android applications to run on Windows 10.

据报道，微软正在创建一个子系统，类似于Linux的Windows子系统，该子系统允许Android应用程序在Windows 10上运行。

With around a 70% mobile market share and, unlike iOS, an open app ecosystem, it would be a mistake not to bring Android support directly into Windows 10.

拥有约70％的移动市场份额，并且与iOS不同，它是一个开放式应用生态系统，因此如果不将Android支持直接引入Windows 10将是一个错误。

Microsoft realizes this and has already started offering limited support for launching Android apps in Windows 10 using the Your Phone app and supported Android devices. Launching Android apps via Your Phone, though, is done by streaming the apps from the phone rather than running them on Windows 10.

Microsoft意识到了这一点，并且已经开始提供有限的支持，以便使用Your Phone应用程序和受支持的Android设备在Windows 10中启动Android应用程序。不过，通过您的手机启动Android应用是通过从手机流式传输应用来完成的，而不是在Windows 10上运行它们。

According to Windows Central's Zac Bowden, Microsoft is working on a new subsystem for Windows 10 that would allow Android apps to be distributed via the Microsoft store and run in a virtualized environment.

根据Windows Central的Zac Bowden的说法，微软正在为Windows 10开发一个新的子系统，该子系统将允许Android应用程序通过Microsoft商店分发并在虚拟化环境中运行。

"Microsoft is working on a software solution that would allow app developers to bring their Android apps to Windows 10 with little to no code changes by packaging them as an MSIX and allowing developers to submit them to the Microsoft Store," Windows Central reported.

Windows Central报道说：“微软正在开发一种软​​件解决方案，该解决方案允许应用程序开发人员将它们打包为MSIX并允许开发人员将其提交到Microsoft Store，从而无需更改代码就可以将其Android应用程序引入Windows 10。

This new initiative is called 'Project Latte,' and similar to Windows Subsystem for Linux (WSL), will create a virtualized Android environment running directly within Windows 10.

这个新计划被称为“ Project Latte”，类似于Linux的Windows子系统（WSL），它将创建直接在Windows 10中运行的虚拟化Android环境。

The biggest obstacle will be the graphical user interface requirement of Android apps, but with the release of WSL 2, Microsoft has begun working on a project called 'WSL-G,' or 'WSL - Graphics Architecture.' This project utilizes a built-in Wayland display server to run Linux GUI apps directly in the Windows 10 user interface.

最大的障碍将是Android应用程序对图形用户界面的要求，但是随着WSL 2的发布，微软已经开始着手开发一个名为``WSL-G''或``WSL-图形架构''的项目。该项目利用内置的Wayland显示服务器直接在Windows 10用户界面中运行Linux GUI应用程序。

An XDC 2020 talk by Microsoft's Steve Pronovost goes into further detail about the new WSL-G feature that Microsoft is creating.

Microsoft的Steve Pronovost在XDC 2020上的演讲进一步详细介绍了Microsoft正在创建的WSL-G新功能。

Porting WSL, including the WSL-G component or something similar, to run the Android operating system would allow Windows 10 to run virtualized Android apps.

将WSL（包括WSL-G组件或类似组件）移植到运行Android操作系统，将允许Windows 10运行虚拟化的Android应用程序。

Bowden states that it is unlikely that this new subsystem will support Google Play as Google does not allow this service to be installed on non-native Android or ChromeOS devices.

Bowden指出，该新子系统不太可能支持Google Play，因为Google不允许在非本机Android或ChromeOS设备上安装此服务。

However, the open architecture of Android apps will allow developers to package them and distribute them through the Microsoft Store. Unfortunately, this means that Microsoft will also have to devote a team to reviewing submitted packages for malware, as malicious Android apps have become a nightmare for Google.

但是，Android应用程序的开放式体系结构将允许开发人员打包它们并通过Microsoft Store分发它们。不幸的是，由于恶意的Android应用程序已成为Google的噩梦，这意味着Microsoft还必须专门组成一个团队来审查所提交的恶意软件包。

BleepingComputer has reached out to Microsoft to learn more about this project but has not heard back as of yet.

BleepingComputer已与Microsoft联系，以了解有关该项目的更多信息，但到目前为止尚未得到回音。
