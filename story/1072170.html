为什么我们从 Google Places 切换到我们的 Autocomplete API

2021-08-09 03:56:13

我的一天过得很顺利，在查看我们在 Stripe 上最新更新的 MRR 后，我心情很好，我决定打开我的电子邮件，看看客户是否有任何问题需要回答。就在那时，我注意到一封来自 Google 的电子邮件，标题为“Google Cloud Platform & APIs: Payment received”，然后我的一天就发生了微妙的变化。我打开电子邮件。 “付款已收到”，它用粗体大写。 “太好了”，一切都很顺利，对吧？错误的。您已于 2021 年 6 月 21 日向 Google Cloud Platform 和 API 支付了 1,000.00 美元。“好吧，这可能只是一个错误”，当我打开 Developer API 控制台时，我心里想。但是，不，你瞧，在打开 15 个不同的账单小节之后，每个小节的描述性名称都比下一个（“成本表”、“成本分解”、“承诺”、“承诺分析”）少，并且没有找到这个简单问题的答案“这个月API调用了多少次？”，我终于看到了我们这个月的消费情况（我写这篇文章的时候试着找回来，后来找不到了） .似乎我们实际上是在为我们消费的东西买单。只有一个小问题：什么？什么？什么？这怎么可能？我仍然不明白，但显然它应该是有道理的：

为什么我们要为大气数据付费，这到底意味着什么？我猜谷歌觉得他们可以在“联系数据”上多花 100 美元，不管那是什么。大气数据 165.01 美元：我一直在使用地点/自动完成功能，为什么我还要为大气数据付费？在“联系数据”上额外投入 99 美元，此时额外的 100 美元是多少？嗯...也许这可以解释为什么 Google 的 API 产品的定价页面如此晦涩。但我最喜欢的部分是我们实际上并没有支付全价，这是我们的折扣价！他们多么慷慨！哦，如果这仍然没有意义，这里为您提供另一种定价明细。是的，在任何地方添加一些下拉菜单，这会让事情变得更清楚：那是我开始寻找替代方案的时候。值得庆幸的是，到 2021 年会有相当数量的这些：Mapbox、OpenCageData、LocationIQ 和 GeoCod.IO 仅举几例。

最后，我选择了 Mapbox：它不是所有选项中最便宜的，但它似乎在可负担性和数据相关性之间取得了良好的平衡，在整个空间的自动完成中获得了最相关的结果。来自谷歌混乱的控制台，他们严肃的仪表板也非常令人耳目一新：现在，我们每个月的前 100,000 次通话都是免费的，我们最多为每额外 1000 次通话支付 0.75 美元。到目前为止，您已经猜到我肯定不会使用任何 Google 的 API，除非对定价和未来的替代方案进行严格审查。希望这篇文章可以帮助更多人在盲目选择默认选项之前三思而后行。脚注：虽然到目前为止我对 Mapbox 的服务和产品非常满意，但本文绝不是 Mapbox 赞助或请求的。我只是一个幻想破灭的谷歌客户，在地狱中找到了一杯水。想要为您的社区建立自己的自定义工作委员会？前往 Niceboard.co 开始您的 7 天免费试用！
