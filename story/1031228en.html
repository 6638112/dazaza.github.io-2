ESP8266-EVB is Evaluation board for ESP8266 with relay, button, UEXT, all GPIOs available on 0.1" header ESP8266-EVB is  OSHW certified Open Source Hardware with UID BG000015

ESP8266-EVB是ESP8266的评估板，带有继电器、按钮、UEXT以及0.1&34；接头上提供的所有GPIO ESP8266-EVB是OSHW认证的开源硬件，UID为BG000015。

ESP8266-EVB is board suitable for resarch and development of espressif's ESP8266EX chip. The module be used for home automation, smart plugs and lights, mesh networks, industrial wireless control, baby monitors, IP cameras, sensor networks, wearable wlectronics, etc. ESP8266EX is very highly integrated SoC which includes Tensilica's L106 32-bit core processor; SRAM; power management unit; RF front end. ESP8266EX has a number of interfaces - I2C, SPI, SDIO and also a number of free GPIO pins. The chip allows the implementation of a WIFI TCP-IP stack with just few extra components beside the ESP8266EX.

ESP8266-EVB是一款适合研发espressif的ESP8266EX芯片的主板。该模块可用于家庭自动化、智能插头和灯、网状网络、工业无线控制、婴儿监视器、IP摄像头、传感器网络、可穿戴电子设备等。ESP8266EX是高度集成的SoC，其中包括Tensilic&39；的L106 32位核心处理器；SRAM；电源管理单元；RF前端。ESP8266EX有多个接口-I2C、SPI、SDIO和一些空闲的GPIO引脚。该芯片除了ESP8266EX外，只需少量额外组件即可实现WiFi TCP-IP堆栈。

Board features: Row of 16 pin holes at 0.1" step with all ESP8266 signals accessible; male-male 16-pin connector included

电路板特征：0.1和#34处有一排16针孔；阶梯状，所有ESP8266信号均可访问；包括凸式16针连接器。

You need to establish UART connection to the board's UEXT (using RX, TX, GND). The best approach is to use a USB to serial cable with 3.3V TTL converter and a terminal program. Then power the board in standard "boot from FLASH mode" (no button pressed during power-up).

您需要建立到电路板UEXT的UART连接(使用RX、TX、GND)。最好的办法是使用带有3.3V TTL转换器和终端程序的USB转串口电缆。然后以标准闪存模式启动电路板(通电期间没有按下按钮)。

I want to send basic AT commands to the board but I receive no response. The strange thing is that I receive "ready" when I power the module. What is the problem?

我想向电路板发送基本的AT命令，但没有收到响应。奇怪的是，当我给模块加电时，我收到了"；Ready"；。有什么问题吗？

All AT commands must end with carriage return and line feed - "/r/n". Your terminal software might have such a new line option - transmitting CR+LF at the end of each command. If it doesn't - either use another terminal software or try to send the commands with "CTRL"+"J" keyboard combination, instead of "ENTER".

所有AT命令必须以回车符和换行符-"；/r/n"；结束。您的终端软件可能有这样一个新的行选项-在每个命令的末尾发送CR+LF。如果没有，请使用其他终端软件或尝试使用"；CTRL&34；+#34；J"；键盘组合发送命令，而不是输入"；。
