Microsoft's Latest Windows 10 Test Builds Includes Promised x64 Arm Emulation
微软最新的Windows 10测试版本包括承诺的x64 Arm仿真

2020-12-11 15:04:29

Microsoft made available two different Windows 10 test builds today, December 10. There's a Dev Channel build 21277 from the pre-release branch which includes the promised x64 app emulation for Arm, among other features. And there's a Windows 10 test build, No. 20277, that's available to the Dev Channel testers from the Iron (Fe) branch available today, as well. The RS_Prerelease build 21277 -- which ultimately is expected to be designated as the "Cobalt" branch -- includes the features Microsoft had previously been testing but removed at the end of October. This includes the updated emoji picker, redesigned touch keyboard, voice typing, theme-aware splash screens and more. It also provides the aforementioned Arm emulation support.

微软今天（12月10日）提供了两个不同的Windows 10测试版本。预发行分支中有一个Dev Channel版本21277，其中包括承诺的用于Arm的x64应用仿真，以及其他功能。 Windows 10测试版本号20277，也可以从Iron（Fe）分支提供给Dev Channel测试人员使用。 RS_Prerelease内部版本21277-最终有望被指定为＆＃34; Cobalt＆＃34;。分支-包括Microsoft先前已测试但在10月底删除的功能。其中包括更新的表情符号选择器，重新设计的触摸键盘，语音输入，可识别主题的启动屏幕等。它还提供了上述Arm仿真支持。

Update: As Microsoft execs point out, the Dev Channel releases are not tied to a specific feature update these days. That technically means Microsoft could take some, all, or none of the features in the Prerelease build and move them into any Windows 10 feature update -- 21H1, 21H2 or later. Currently, Windows on Arm natively supports Arm apps, including ARM64 versions. But so far, only 32-bit Intel (x86) apps are supported in emulation. This lack of x64 emulation has limited the number of apps that can run on Windows on Arm devices, since apps that are 64-bit only have only been available on Windows on Arm (WoA) devices if and when developers created native versions of them. As of now, these x64 Arm apps also can run in emulation.

更新：正如Microsoft高管指出的那样，如今开发通道版本与特定功能更新无关。从技术上讲，这意味着Microsoft可以采用Prerelease构建中的部分，全部或不提供任何功能，并将其移至任何Windows 10功能更新中（21H1、21H2或更高版本）。当前，Windows on Arm本机支持Arm应用程序，包括ARM64版本。但是到目前为止，仿真中仅支持32位Intel（x86）应用程序。由于缺乏x64仿真，因此只有在开发人员创建本机版本时以及在Windows on Arm（WoA）设备上才只能使用64位应用程序，因为它们只能在Arm设备上的Windows上运行。到目前为止，这些x64 Arm应用程序也可以在仿真中运行。

Today's Iron release (20277) includes a couple fixes and several Known Issues. The Pre-Release build includes a bunch of new and updated features, fixes and known issues.

今天的Iron版本（20277）包含一些修复程序和一些已知问题。发行前版本包括许多新功能和更新功能，修复程序和已知问题。

Before Insiders rush off to install the RS_Prerelease branch build 20277, they should read the fine print. These builds are currently offered as optional updates to "seekers," meaning Insiders will have to go and choose to install them. Once they do this, they can't go back to Iron; they'll continue to receive RS-Prerelease builds, going forward. Those who don't want to do this can continue to automatically get builds from the Iron branch until Microsoft moves all Insiders to the Dev Channel to the RS-Prerelease builds "some time after the holidays."

Microsoft officials still have not said publicly whether there will be a 21H1 Windows 10 feature update and whether that update will be from the Iron branch. Word is 21H1 will be a very small release (feature-wise) if and when it happens next spring. The 21H2/Cobalt release is shaping up to be a bigger, more feature-rich release, sources have said.
