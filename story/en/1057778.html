Illegal Prime Numbers
非法素数

Jump to navigation Jump to search An illegal prime is a prime number that represents information whose possession or distribution is forbidden in some legal jurisdictions. One of the first illegal primes was found in 2001. When interpreted in a particular way, it describes a computer program that bypasses the digital rights management scheme used on DVDs. Distribution of such a program in the United States is illegal under the Digital Millennium Copyright Act. [1] An illegal prime is a kind of illegal number.

One of the earliest illegal prime numbers was generated in March 2001 by Phil Carmody. Its binary representation corresponds to a compressed version of the C source code of a computer program implementing the DeCSS decryption algorithm, which can be used by a computer to circumvent a DVD's copy protection. [1]

最早的非法素数是菲尔卡托于2001年3月产生的。其二进制表示对应于实现DECS解密算法的计算机程序的C源代码的压缩版本，该计算机程序可以由计算机用于绕过DVD＆＃39; S复制保护。 [1]

Protests against the indictment of DeCSS author Jon Lech Johansen and legislation prohibiting publication of DeCSS code took many forms. [2] One of them was the representation of the illegal code in a form that had an intrinsically archivable quality. Since the bits making up a computer program also represent a number, the plan was for the number to have some special property that would make it archivable and publishable (one method was to print it on a T-shirt). The primality of a number is a fundamental property of number theory and is therefore not dependent on legal definitions of any particular jurisdiction.

抗议DECS的起诉作者作者Jon Lech Johansen和禁止公布DECS守则的立法采取了许多形式。其中一个是非法代码的表现形式，该形式具有内在的存档质量。由于构成计算机程序的比特也代表了一个数字，因此该计划是为了具有一些特殊属性的数字，可以使其归档和可发挥（一种方法是在T恤上打印它）。数字的原始性是数字理论的基本属性，因此不依赖于任何特定管辖权的法律定义。

The large prime database of The Prime Pages website records the top 20 primes of various special forms; one of them is proof of primality using the elliptic curve primality proving (ECPP) algorithm. Thus, if the number were large enough and proved prime using ECPP, it would be published.

Prime页面网站的大型素数数据库记录了各种特殊形式的前20个素数;其中一个是使用椭圆曲线原始证明（ECPP）算法的原始原因。因此，如果数量足够大并使用ecpp证明素数，则将发布。

Specifically, Carmody applied Dirichlet's theorem to several prime candidates of the form k·256 n + b, where k was the decimal representation of the original compressed file. Multiplying by a power of 256 adds as many trailing null characters to the gzip file as indicated in the exponent which would still result in the DeCSS C code when unzipped.

具体而言，Carmody将Dirichlet＆＃39; S定理应用于k·256 n + b形式的几个候选，其中k是原始压缩文件的小数表示。乘以256的力量为GZIP文件添加到GZIP文件中的许多尾随空字符，如指令中所示，仍将导致DECSS C代码在解压缩时。

Of those prime candidates, several were identified as probable prime using the open source program OpenPFGW, and one of them was proved prime using the ECPP algorithm implemented by the Titanix software. [3] [4] Even at the time of discovery in 2001, this 1401-digit number, of the form k·256 2 + 2083, was too small to be mentioned, so Carmody discovered a 1905-digit prime, of the form k·256 211 + 99, that was the tenth largest prime found using ECPP, a remarkable achievement by itself and worthy of being published on the lists of the highest prime numbers. [1] In a way, by having this number independently published for a completely unrelated reason to the DeCSS code, he had been able to evade legal responsibility for the original software.

在那些主要候选者中，使用开源程序OpenPfgw确定几个可能的原始素数，并且使用Titanix软件实现的ECPP算法证明了其中一个。 [3] [4]即使在2001年的发现时，k·256 2 + 2083的这种1401位数字也太小了，所以Carmody发现了一个1905位的素数K·256 211 + 99，这是使用ecpp发现的第十大的主要原子，这本身是一个显着的成就，值得发表在最高素数的名单上。 [1]以一种方式，通过独立于DECS代码完全不相关的原因独立发布的，他能够逃避原始软件的法律责任。

Following this, Carmody discovered an 1811-digit prime—this one being non-compressed, directly executable machine language in the ELF format for Linux i386, implementing the same DeCSS functionality. [5]

在此之后，Carmody发现了一个1811位的Prime-这个是一个非压缩的，直接可执行的机器语言，用于Linux I386的ELF格式，实现相同的DECS功能。 [5]
