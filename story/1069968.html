Sedna 为一个平台提供 3400 万美元的资金，该平台解析大量电子邮件和聊天以自动执行其中的操作项

2021-07-27 15:15:15

B 轮融资由 Insight Partners 领投，Stride.VC、Chalfen Ventures 和 SAP.iO 基金（SAP 的一部分）也参与其中。这笔资金将用于继续围绕 Sedna 的核心功能构建更多数据科学，目的是随着时间的推移进入更广泛的垂直领域。目前其主要业务在供应链参与者领域，其客户包括嘉能可、诺顿和邦吉。金融等领域的其他客户包括新银行 Starling。总部位于伦敦的 Sedna 没有透露估值。 Sedna 的首席执行官兼创始人比尔·多比 (Bill Dobie) 表示，公司的想法是根据他自己的经验产生的。 “我花了数年时间构建软件来帮助用户提高工作效率，但无论我们构建什么，我们都没有真正减少人们的工作量。”原因是：被称为电子邮件的磨盘，其源源不断、未经请求的入站消息，其中一些（刚好足以不容忽视）可能很重要。 “真正让我印象深刻的是将项目移出和移入电子邮件所花费的时间，”他谈到由此产生的“待办事项”时说。因此，Sedna 旨在“阅读”电子邮件并为它们提供更多背景和方向。当涉及到人们的收件箱时，它的系统会删除重复的操作项并从本质上提高点击率：里面的内容更有可能是您真正需要看到的内容。它以非常快的速度完成。 “我们的主要价值在于我们经营的庞大规模，”多比说。 “我们在亚秒级响应时间内阅读了数百万甚至数十亿条消息。”事实上，虽然我们中的许多人没有收到“数百万”电子邮件，但有一个消息世界需要阅读更多。例如，想想基于物联网的诊断将产生的数据量。 “智能”收件箱肯定已经成为消费者的东西——尽管可以说没有一个像你希望的那样工作。 Sedna 值得注意的是它如何将其特定算法调整到特定的垂直领域，让他们在特定组织的内容和工作实践方面变得更加智能。目前，这项工作是由 API 框架驱动的，其中包含“低代码”格式的元素，让人们可以塑造自己的 Sedna 体验。随着时间的推移，我们的目标是让这一切变得更加容易。现在是一个 API 驱动的框架，我们正在使用一些低代码，但主要是它的 SAP 或运输或交易系统，它了解正在进行的交易，然后 Sedna 使用决策树进行分类。 Sedna 可能增长的另一个领域是它如何处理它摄取的信息。目前，该公司的技术可以由客户互连，然后将某些工作移交给 RPA 系统以及特定人员。在那里开发一些第二阶段的软件有一条明显的途径——或者，这表明像 Sedna 这样的东西可能会被一个大型 RPA 玩家抢购或复制。

Insight Partners 的负责人 Rebecca Liu-Doyle 在一份声明中说：“Bill 开始重新构想电子邮件最容易损坏、因此最难修复的地方——管理大量数据和复杂流程的大型团队。” “今天，Sedna 的强大之处在于它能够为任何收件箱体验带来极大的速度、简单性和愉悦感，无论规模或复杂程度如何。我们很高兴与 Sedna 团队合作，因为他们将继续为全球供应链及其他领域的团队提供更智能的数字通信。” Liu-Doyle 在这轮融资中加入了董事会。 SAP 是这一轮的战略投资者，因为 Sedna 有可能帮助其客户在使用 SAP 系统时提高生产力。 “SAP 继续与 SEDNA 合作，为 SAP 客户创造价值。 SAP 全球运输解决方案负责人 Stefan Sauer 在一份声明中表示，将复杂信息转化为更简单的智能协作的能力已成为许多 SAP 客户日益重视的问题。
