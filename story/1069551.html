如今，设置自己的 WiFi 网络非常容易。您在线订购路由器，将其插入电源插座，定义密码，然后就可以开始使用了。 WiFi 快速、可靠且易于使用。但是，如果您想覆盖更广的区域或连接数百个小型设备，它很快就会变得效率低下且成本高昂。去当地的移动网络运营商那里签合同是唯一的方法吗？不！得益于开源技术，您可以构建自己的 LTE 或 5G 网络。 LTE 或 5G 等移动技术为您的专用网络提供了强大的功能。您可以使用少量设备覆盖大片区域。此外，选择正确的频率，您可以通过墙壁或地下获得出色的性能。移动网络比 WiFi 安全得多。更不用说，它现在使您能够获得只有运营商才有的所有功能和功能，例如具有低于 1 米精度的用户设备 (UE) 位置跟踪或高级信令和遥测。您可能害怕开始构建 LTE 或 5G 网络所需的高昂前期成本。幸运的是，情况并没有那么糟糕。最低启动工具包是 LimeSDR mini 和 Raspberry Pi 4。总成本低于 300 美元（假设您已经拥有手机……）！如果您想使用更高级的功能，例如 Carrier Aggregation，实现最高速度并连接数百个 IoT 设备，您需要花费大约 10k$ 购买 Ettus x310 和一些不错的 X86 服务器进行计算。作为软件堆栈，我可以推荐 Ubuntu + srsRAN https://github.com/srsran/srsRAN。许多公司需要为其数据和语音用例提供安全、快速和自动化的网络，这些网络能够覆盖大面积、在建筑物内或地下隧道内可靠。私有 LTE 和 5G 网络满足了这种需求，如果它们基于开放技术，也可以提供很好的性价比。它们在许多行业实施：校园——大学、医院或大办公室。他们都有大量的人和设备需要连接。专用移动网络为大学提供了成本效益和绝佳的学习机会。

石油和天然气——这些环境恶劣，充满工业物联网，在地下运行，需要 LTE 和 5G 提供的可靠性和安全性。军事——私有 5G 网络是现代互联军事行动的关键。传感器和无人机的态势感知是一种力量倍增器。但是，它需要只有最新的移动技术才能提供的性能和安全性。制造——梅赛德斯奔驰和波什已经使用私有 5G 网络和 MEC 跟踪生产线并优化他们的制造流程。必须迅速做出决定，并当场做出决定。将数据发送到公共云太慢，本地微云在此类用例中显示出更好的结果。用户设备——从手机到物联网设备、无人机或自动驾驶汽车，无所不包。无线电接入网络——这是包含基站、天线和 L1/L2 处理的部分。 OpenRAN 用于分解和开放的无线电接入网络，在 Ubuntu 操作系统 Magma 核心上运行 RIC、RU、CU 和 DU，提供最先进的 EPC，具有联合网关和接入网关（自 1.15 迁移到 Ubuntu）

MAAS、LXD 和 MicroK8s 的组合构建了最先进的微云，允许您将边缘应用程序作为 VM、容器、裸机运行，并提供所有 EPA 功能，如 DPDK、SR-IOV 或 NVIDIA GPU 加速在设计时考虑到经济性， Canonical 的电信解决方案可确保投资回报率，同时提供一流的质量。通过以智能方式运行基础设施和应用程序来节省成本，确保从第 0 天到第 N 天完全自动化。
