A repository of FreeDOS-compatible freeware and OSS software
与FreeDOS兼容的免费软件和OSS软件的存储库

2022-02-22 17:59:35

This is a repository of FreeDOS-compatible freeware and OSS software. Not “abandonware”, by which I mean that you will not find formerly commercial software here that have not been explicitly made open-source or at least freeware.

这是一个与FreeDOS兼容的免费软件和OSS软件的存储库。不是“放弃软件”，我的意思是，你在这里找不到以前的商业软件，这些软件没有明确的开源或至少是免费的。

However, non-crippled shareware with a single nag screen at the beginning or end is fair game. Whether you are going to try to send money to a developer from thirty or forty years ago who could be dead by now is up to you.

然而，在开始或结束时只有一个唠叨屏幕的非残废共享软件是公平的。你是否打算把钱寄给一个三十年前或四十年前可能已经去世的开发商，这取决于你。

Having said all that, there are a lot of little utilities from a kinder, gentler age, when people would just make their stuff available without worrying about licenses. These will be regarded as public domain until someone informs me otherwise.

话虽如此，在一个更友善、更温和的时代，有很多小的公用事业，人们只需要提供他们的东西，而不用担心许可证。除非有人通知我，否则这些将被视为公共领域。

This is not meant to be a conclusive collection of DOS applications. These are apps that I have tested on FreeDOS 1.2 (on bare silicon and in the DOSBox emulator) and that I find useful, fun or interesting. If you need something more comprehensive, try this site or this one. I am not a gamer at all, so if that is what you are looking for, well, sorry, I only have a few of the simplest ones.

这并不是DOS应用程序的最终集合。这些是我在FreeDOS 1.2（裸硅和DOSBox模拟器）上测试过的应用程序，我觉得它们有用、有趣或有趣。如果你需要更全面的东西，试试这个网站或这个。我根本不是一个游戏玩家，所以如果这就是你想要的，那么，对不起，我只有几个最简单的。

An example: FreeDOS comes with DosZip Commander. I use it all the time. And if you don’t like that one, it also has DOS Navigator. There are dozens of Norton Commander clones out there: Directory Maven, Volkov Commander, you name it. But generally one is only going to end up in this repository if it offers a clear difference to and advantage over Doszip Commander.

例如：FreeDOS附带DosZip Commander。我一直在用它。如果你不喜欢这个，它也有DOS导航器。那里有几十个诺顿指挥官克隆人：目录马文，沃尔科夫指挥官，随便你怎么说。但一般来说，只有在Doszip Commander与Doszip Commander有明显区别和优势的情况下，才会出现在这个存储库中。

Since I started this site, some apps have been incorporated into the FreeDOS installation itself. I’ll weed them out if I see that happening. If in doubt, use the one that comes with FreeDOS.

自从我创建这个网站以来，一些应用已经被整合到FreeDOS安装中。如果我看到这种情况，我会把他们除掉。如果有疑问，请使用FreeDOS附带的软件。

I also slip in a few little apps that I wrote myself. You will know them by a certain primitive look and feel :-)

我还加入了一些我自己编写的小应用程序。你会通过某种原始的外观和感觉来了解它们：-）

Download the package, transfer it to your FreeDOS system and extract the zip file to the ROOT directory (C:\).

下载软件包，将其传输到FreeDOS系统，并将zip文件解压缩到根目录（C:\）。

You might be tempted to use the DosZip Commander, which is bundled with FreeDOS and can be called up with the command dz, or similar utilities. I advise against this. These packages were actually created under Linux and you may come across weird crc errors if you use dz or similar apps. The command line works fine.

您可能很想使用DosZip Commander，它与FreeDOS捆绑在一起，可以通过命令dz或类似的实用程序调用。我建议不要这样做。这些软件包实际上是在Linux下创建的，如果你使用dz或类似的应用程序，你可能会遇到奇怪的crc错误。命令行运行良好。

It is important to extract these packages to the root directory!. There is an actual reason for this. Some of these very old programs are not relocatable: They have hardcoded paths inside them and will crash or refuse to load if you try to run them from a strange directory. I have tried really hard to stick to the FreeDOS file system and not litter your root with directories, but sometimes I simply can’t. Better then, to have a single installation procedure for all my packages.

将这些包解压缩到根目录！非常重要！。这是有实际原因的。其中一些非常古老的程序是不可重新定位的：它们内部有硬编码的路径，如果您试图从一个陌生的目录运行它们，它们会崩溃或拒绝加载。我非常努力地坚持使用FreeDOS文件系统，不在根目录中乱放目录，但有时我就是做不到。最好是对我的所有软件包都有一个单一的安装过程。

Programs that do not run directly from c:\FDOS\BIN will have a convenient batch file placed in C:\FDOS\LINKS. If a program requires changes to CONFIG.SYS or AUTOEXEC.BAT, that will be documented here.

不直接从c:\FDOS\BIN运行的程序将在c:\FDOS\LINKS中放置一个方便的批处理文件。如果程序需要更改配置。SYS或AUTOEXEC。蝙蝠，这将被记录在这里。

Sorry, there is no uninstall facility. But most of these packages consist of a single directory in C:\FDOS\PROGS (or C:\FDOS\DEVEL), a batch file in C:\FDOS\LINKS and an information file in C:\FDOS\APPINFO, so it shouldn’t be too hard.

抱歉，没有卸载功能。但是，这些包中的大多数都包含一个位于C:\FDOS\PROGS（或C:\FDOS\DEVEL）中的目录、一个位于C:\FDOS\LINKS中的批处理文件和一个位于C:\FDOS\APPINFO中的信息文件，所以这应该不会太难。
