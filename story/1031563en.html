However, I would think introducing a ~ git lfs smudge --usehardlink fileABC may also good for people like me still on the NTFS drive, and Linux Filesystem... I had design the "git filter process" that cheat the git by first give it a same content as "clean" filter. While ran a child process to actually smudge files at the end of the filter process. It is cheating, and made git confuse a little bit but no thing major. This hack can be apply for hardlink too. I can do this perl filter script but it would be a deep hack without some thing like git lfs smudge --useharklink fileABC

然而，我认为引入~git lfs污点--usehardlink fileABC对像我这样仍在NTFS驱动器和Linux文件系统上的人也有好处。我设计了git筛选器进程，它欺骗了git，首先给它一个与清除筛选器相同的内容。同时运行子进程以在过滤进程结束时实际涂抹文件。这是作弊，让git有点迷惑，但没什么大不了的。这个黑客也可以申请硬链接。我可以使用这个Perl过滤器脚本，但如果没有git lfs污点之类的东西--useharklink fileABC--这将是一次深度黑客攻击
