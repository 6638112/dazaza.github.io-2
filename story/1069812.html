在今年早些时候以 5500 万美元收购 SignRequest 之后，Box 推出了 Box Sign，它允许用户以电子方式签署合同并请求签名

2021-07-26 23:29:30

Box Inc. 正在通过添加一项重要的新功能来构建其内容云。该公司今天表示，它正在推出 Box Sign，这是 Box 中的一种原生电子签名功能，将使用户能够以电子方式签署合同并请求他人签名。 Box 表示，Box Sign 支持无限签名，并带有一组强大的应用程序编程接口，使企业能够完全在云中实现协议的现代化和管理。 Box Sign 功能是通过该公司今年早些时候以 5500 万美元收购荷兰初创公司 SignRequest BV 而实现的。 Box 首席执行官 Aaron Levie 在接受采访时告诉 SiliconANGLE，在 COVID-19 大流行期间，电子签名是其客户最需要的功能。由于大多数企业被迫在线开展业务，人们无法再面对面地签署合同和交易。 “越来越多的交易正在从纸质流程转移到云端，”Levie 说。 “但是现有的解决方案存在问题。它们成本太高，分散且存在安全问题。” Box Sign 旨在解决所有三个问题。它与 Box 原生集成，这意味着不再存在碎片化，客户无需支付额外费用即可使用它。这些功能甚至可以通过 Box for Salesforce 集成提供，因此可以从 Salesforce 内部生成和创建保密协议、合同等。该公司表示，至于安全性，Box Sign 继承了 Box 坚如磐石的安全性和合规性。

“这是我们多年来所做的最重要的新产品，”Levie 说。 Box Sign 功能包括向任何人发送和请求签名的能力，无论他们是否使用 Box，用于常见可重复流程（如 NDA）的模板，以及跟踪工具和电子邮件提醒和通知。 Box 表示，从今天开始，部分客户可以使用 Box Sign，其他用户将在未来几个月内看到它。 ZK Research 首席分析师 Zeus Kerravala 表示：“Box 的方法不同，因为它考虑了电子签名的生命周期，其中包括实际签名以及上游和下游流程。” “随着企业采用电子签名，重要的是将其视为整个文档管理流程的一部分，而不是孤立地完成。” Box Sign 功能帮助 Box 构建它喜欢称之为“内容云”的东西，它可以被认为是管理所有企业文档和文书工作的基础设施。借助 Box Content Cloud，该公司旨在通过增加关键自动化和安全功能的多合一服务取代分散的遗留系统，例如网络存储和文档和文件管理工具。 “我们正在构建一个内容云，帮助客户在单一架构中管理内容的整个生命周期，”Levie 说。 “所以你会看到我们投资于所有这些支柱。”为此，Box 宣布了一项名为 Enterprise Plus 的新订阅计划，该计划从今天开始可用。 Levie 说，它是迄今为止最高的级别，包括所有主要的附加组件。它将 Box 的基本功能与 Box Shield、Box Governance、Box Relay、Box Platform 和新的 Box Sign 捆绑在一起。除此之外，Enterprise Plus 客户将能够直接从 Salesforce 内发送无限数量的文档进行签名。

Levie 表示，他认为 Box Enterprise 可能会看到一些强劲的动力，并指出该公司最近看到了自去年第四季度以来的“增长重新加速”，超过 10 万美元的交易显着增长。他还指出了混合工作的趋势，在这种趋势中，人们将继续花费大量时间远程工作，而在办公室的时间有限。 “新的病毒变种和疫苗接种不足意味着混合工作将持续很长一段时间，”Levie 说。 “我们认为内容在混合工作中扮演着如此重要的角色，它将成为未来混合工作的主要组成部分。”加入我们的 Cube 俱乐部和 Cube 活动专家社区，表达您对我们使命的支持。加入社区，其中包括 Amazon Web Services，即将成为 Amazon.com 首席执行官 Andy Jassy、戴尔科技公司创始人兼首席执行官迈克尔戴尔、英特尔首席执行官 Pat Gelsinger 以及更多名人和专家。我们将于 6 月 16 日举行第二次云创业展示会。单击此处加入免费且开放的 Startup Showcase 活动。
