Google penalizes you for using Google Analytics

2021-05-05 20:28:05

Google is adding a page experience update to their search result ranking. They recently announced it would roll out in mid-June and finished at the end of August. In this blog post, we will explain what that entails and that Simple Analytics has your back.

谷歌正在添加页面体验更新到他们的搜索结果排名。他们最近宣布将在6月中旬推出并在8月底完成。在这个博客文章中，我们将解释它需要的东西，并且简单的分析有您的背部。

We don’t go too deep into these metrics, but the general message is that they are all calculated within the performance score of Google Lighthouse. Lighthouse is a tool that Google created to measure the performance of a website. It gives a score to 4 categories: performance, accessibility, best practices, and SEO. The Core Web Vitals are included in this first category: the performance score. You can try it out yourself at web.dev/measure (Google’s online interface for Lighthouse). It runs on the servers of Google.

我们不会太深入了解这些指标，但常规消息是它们都在谷歌灯塔的性能得分内计算。灯塔是一个谷歌创建来衡量网站性能的工具。它给出了4个类别的分数：性能，可访问性，最佳实践和SEO。核心网络生命值包含在本第一个类别中：性能分数。您可以在web.dev/measure中尝试自己（谷歌的灯塔的在线界面）。它在谷歌的服务器上运行。

To learn more about Core Web Vitals, you should check out Ahrefs blog post. It goes in-depth about those three metrics.

要了解有关核心Web Vitals的更多信息，您应该查看Ahrefs博客文章。它深入了解这三项指标。

Google will begin using page experience as part of their ranking systems starting in mid-June 2021. However, page experience won’t play its full role as part of those systems until the end of August.

谷歌将开始使用页面体验，作为从6月2021年中期开始的排名系统的一部分。然而，页面体验不会在8月底之前作为这些系统的一部分发挥其全部作用。

Analytics tools do impact performances on websites. They usually add a script to the website that adds functionality. Logically this has a cost in terms of performance. Some analytics businesses care a lot about this, and others don’t have that high priority. At Simple Analytics, we care a lot about the impact of our embedded script on our customers’ websites. We don’t want to load big scripts into the browser of our customers. Although this all sounds nice, we should prove it to you as well.

Analytics工具在网站上进行影响性能。它们通常将脚本添加到添加功能的网站。逻辑上，这在性能方面具有成本。一些分析企业的企业关心这一点，其他人没有那么高的优先级。在简单的分析中，我们关心我们的嵌入式脚本对客户网站的影响。我们不想将大脚本加载到客户的浏览器中。虽然这一切都听起来不错，但我们也应该证明你。

Olav Pekeberg compared his website with and without Google Analytics. He found that he got a performance score boost from 93 (with GA) to 100 (without GA). The score goes from 1 to 100, and the higher, the better. He asked us how Simple Analytics was performing compared to Google Analytics. We should perform better based on all the actions we took but were triggered to prove ourselves.

Olav Pekeberg将他的网站与没有Google Analytics的网站进行了比较。他发现他从93（带Ga）到100（没有Ga），他的性能得分提升。得分从1到100岁，更高，更好。他向我们询问了与Google Analytics相比的简单分析是如何进行的。我们应该根据我们所采取的所有行动来表现更好，但被触发以证明自己。

We started with running a few tests on our computer, but the results always varied. We couldn’t get a consistent result. This could have been because of the internet connection or other factors we don’t know about. Then we did run it Google’s online interface for Lighthouse. Every time we did run the test, it showed us a different result. As we wanted to know if we performed better, we needed to run it a lot.

我们开始在计算机上运行几个测试，但结果始终变化。我们无法获得一致的结果。这可能是因为我们不了解的互联网连接或其他因素。然后我们确实运行了Google的灯塔在线界面。每次我们都经过测试时，它向我们展示了一个不同的结果。正如我们想知道我们是否表现更好，我们需要运行很多。

Luckily Dan Sloan suggested that we run Lighthouse on a fresh VPS using the command line tool and run it a few times, grab the data via JSON, then compare without SA. That will give you the best sample set without other interfering factors.

幸运的是，丹斯斯隆建议我们使用命令行工具在新鲜VPS上运行灯塔并运行几次，通过JSON获取数据，然后在没有SA的情况下比较。这将为您提供最好的样本集，没有其他干扰因素。

We followed Dan’s advice and created a little Node.js script around Lighthouse. We did run this test 500 times to get a trustworthy average.

我们遵循DAN的建议，并在灯塔周围创建了一个小节点.JS脚本。我们确实运行此测试500次以获得可靠的平均水平。

We created three pages on our blog; one with no analytics script, one with Simple Analytics, and one with Google Analytics. All are using the latest versions with default configurations. The test pages are hidden for our blog visitors, so we don’t accidentally send data to Google.

我们在我们的博客上创建了三页;一个没有分析脚本，一个带有简单的分析，以及一个带Google Analytics的脚本。所有这些都使用最新版本，默认配置。测试页面隐藏于我们的博客访问者，因此我们不会意外将数据发送到Google。

The test tool ran around 2 hours to complete because every test did run synchronously. We didn’t want to run them in parallel as this could interfere with the test results.

测试工具在2小时内运行才能完成，因为每个测试都同步运行。我们不想并行运行它们，因为这可能会干扰测试结果。

As you can see, we don’t impact the performance score with Simple Analytics at all. But don’t take our word for it. We made our test public, so everybody can run the test independently. Please share your results.

正如您所看到的，我们根本不会影响简单的分析。但不要把我们的话语带来。我们宣传了我们的测试，所以每个人都可以独立运行测试。请分享您的结果。

He did run the tests both on Lighthouse and Google Page Speed. Both systems check for the Core Web Vitals. We rerun the test because 10 can be assumed a low number. We did run the test ourselves on his website as well. We ran that test 100 times.

他确实在灯塔和谷歌页面速度上进行了测试。两个系统都检查核心网络威力。我们重新运行了测试，因为10可以假设低数量。我们确实在他的网站上进行了测试。我们跑了100次测试。

There is a common misconception around asynchronous code. People believe there is no impact on the performance score. This belief is not valid. We did run our test with scripts running with the async attribute. Simple Analytics did support this from the start; Google Analytics followed by making it part of their latest embed script. It affects the performance score, therefore, on the page ranking in Google as well.

异步代码存在常见的误解。人们认为对绩效得分没有影响。这种信念无效。我们确实使用使用Async属性运行的脚本进行测试。简单的分析确实支持这一点; Google Analytics随后使其成为他们最新的嵌入脚本的一部分。因此，它影响谷歌排名的性能分数。

As you can see in the above numbers, Simple Analytics is performing better than Google Analytics. Google hurts the performance score by roughly 4%, and we almost have zero impact (0.1%).

正如您在上面的数字中看到的那样，简单的分析比Google Analytics更好。谷歌伤害了性能得分大约4％，我们几乎具有零影响（0.1％）。

If you care about a fast website and are still using Google Analytics, this could be a good moment to switch. Do you want to score high on SERP (Search Engine Results Page), then it could have an impact, but it’s unknown how much. They write on their blog post:

如果您关心快速网站并仍在使用Google Analytics，这可能是切换的好时刻。你想在SERP上得分高（搜索引擎结果页面），然后它可能会产生影响，但它是未知的。他们在他们的博客文章上写道：

…, while this update is designed to highlight pages that offer great user experiences, page experience remains one of many factors our systems take into account. Given this, sites generally should not expect drastic changes. In addition, because we’re doing this as a gradual rollout, we will be able to monitor for any unexpected or unintended issues.

......，此更新旨在突出显示提供优秀用户体验的页面，页面体验仍然是我们的系统考虑到的许多因素之一。鉴于此，网站通常不应期望发生剧烈的变化。此外，因为我们这样做是渐进的推出，我们将能够监控任何意外或意外的问题。
