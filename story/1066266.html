GitHub桌面2.9包括挤压，重新排序，修改等等

2021-06-17 03:13:12

在GitHub桌面2.7中，我们发布了樱桃挑选和介绍了拖放，在GitHub桌面2.8中我们对差异进行了几个改进。我们继续势头和扩展拖放，以允许您在历史记录中进行挤压和重新排序，修改以前的提交，从早期提交中启动新分支机构等。 GitHub Desktop现在有更多的Git，让您专注于重要的事情。

许多开发人员深入关注他们的承诺历史，并用它来讲述关于他们项目进展的一致故事。如果一组提交代表单个工作单位，或者项目要求每个拉请求只有一个提交，只需将它们拖动到另一个提交以将它们挤压在一起并添加一个捕获整个图片的新提交消息。

合并后，您也可以选择南瓜和rebase作为合并的一部分。

如果提交的方式是以差异且难以遵循的方式命令，现在您只需在您的历史记录中拖放它们即可拖放它们。

多次，您可能需要查看旧版本的项目，以调查错误或在最新版本之上创建修补程序。现在在桌面中，您可以从您的历史记录中的任何提交中查看新分支，允许您在那个时间点轻松查看回购状态，同时不会对您正在进行的分支产生负面影响。

GitHub Desktop长期包含了撤消您的上次提交的能力将所有更改放回工作目录中。如果您的工作目录有很多现有的变化，并且只想才能进行小变化，撤消可能并不符合很大的契合。现在在您的提交历史记录中，您可以选择修改上次提交以更新提交消息或向提交添加更改。

最后，随着当今的发布，使用新的M1芯片的Apple硅机上的用户将升级到GitHub桌面的本机构建，提高性能和减少崩溃。我们希望为@denisameling延伸巨大的谢谢，以便他的工作在帮助中实现现实。
