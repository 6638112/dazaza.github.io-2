谷歌正在修复 Chrome OS 91 中的一个错误，该错误显然是由一个错字引起的，该错误在重新启动时将 Chromebook 所有者锁定在设备之外

2021-07-23 19:00:07

在 Chrome OS 91 方面，谷歌似乎无法休息。首先，我们看到许多用户报告他们的设备在升级到 91.0.4472.147 后使用了大量的 CPU。虽然谷歌此后不久取消了更新并将所有人回滚到 91.0.4472.114，但这设法锁定了 Linux 应用程序。现在我们看到 91.0.4772.165 的到来，此更新引入了一个可怕的错误，该错误正在左右破坏 Chromebook。 ChromeOS subreddit 的一位版主最近向我们透露了这个新的 Chrome 操作系统更新，它将用户锁定在他们的 Chromebook 之外。如果您在系统托盘中收到更新计算机的提示，请不要关闭 Chromebook。否则 Chrome OS 会在重新启动时自动安装它，这意味着您的设备将卡在有问题的 91.0.4772.165 版本中。虽然谷歌意识到了这个问题并正在从其服务器中提取更新，但损害已经造成。一位 redditor 报告说，他们的两台 Chromebook 将不再在登录屏幕上接受他们的密码——他们的一台笔记本电脑开始启动循环。即使是 powerwash 也无法挽救其他用户的 Asus Chromebook C436，迫使他们使用恢复 USB 来恢复并运行他们的 USB。拙劣的 91.0.4772.165 更新似乎导致了重要文件的大量丢失，很多人都很生气。我们不确定这个主要错误是如何通过三个开发者渠道进入稳定版的，我们希望你们中的很多人都不会受到它的影响。整个事件凸显了谷歌对其 Chrome OS 版本缺乏质量控制——对于计划长期使用 Chrome OS 的新用户来说，这当然不是一个好主意。在用其拙劣的版本破坏了一堆 Chromebook 之后，谷歌从其更新矩阵中删除了 91.0.4472.165 版本，将 Chrome 设备回滚到 91.0.4472.147。由于最近的零日漏洞利用，它不会那么安全，但至少您将能够访问您的笔记本电脑。谷歌澄清说，该团队发现了问题，并将于明天，即 7 月 21 日发布修复程序。对于担心数据丢失的受影响用户来说，还有一个好消息：如果你等到周三，你将能够再次登录，而无需进行 powerwash当您的设备自动应用更新时（您可以通过进入访客模式手动触发它）。谷歌将在 7 月 21 日推出更新的 Chrome OS 91 版本并进行修复。那么发生了什么？感谢 Reddit 上一位目光敏锐的用户的工作，我们现在知道一个错字似乎是导致如此多用户无法使用 Chromebook 的原因。通过查看此文件中的差异，我们可以看到 Google 忘记在条件语句中添加第二个“&”，以防止 Chrome OS 解密您的登录信息（登录时需要）。

虽然我们很高兴谷歌迅速采取行动并将拙劣的更新从其服务器上撤下，但损害已经造成。当然，您应该定期创建重要文件的备份，但您的 Chromebook 也不应该任意强迫您删除本地文件。我无法理解使用 Chromebook 的企业——甚至大学——正在经历什么。管理员可以一次远程批量更新数千台受管理的 Chromebook。对昨天的灾难进行故障排除一定很糟糕，而且没有办法补偿最终用户丢失的数据（以及恢复一切所需的时间）。这个问题无疑会给受影响的用户留下伤疤，我希望谷歌采取措施防止此类问题再次发生。
