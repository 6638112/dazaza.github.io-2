COBOL在x86 1.1上为Linux为IBM COBOL编译器系列的最新添加，其中包括Z / OS和AIX的COBOL的企业COBOL。 X86上的Linux Cobol是建设和现代COBOL应用的生产和强大的开发环境。它包括优化COBOL编译器和COBOL运行时库。 X86上的COBOL基于与Z / OS的企业COBOL相同的高级优化技术。它为在X86系统上开发Linux的业务关键COBOL应用程序提供性能和编程功能。 X86上的COBOL型Linux旨在支持客户在云之旅中。它使客户端能够将以COBOL编写的业务关键应用程序战略地部署到混合云环境或最佳拟合平台，包括IBMZ®（Z / OS），IBMPower®系统（AIX）和X86（Linux）平台。

COBOL在X86上为Linux进行支持，支持集成CICS®转换器来处理CICS COBOL应用程序中的本机COBOL和嵌入式CICS语句。它使COBOL批处理应用程序能够访问存储在DB2和CICS TX SFS中的VSAM仿真文件。 COBOL在X86上与IBM TXSeries for Multiplatforms的COBOL与Z / OS的Z / OS和IBM CICS事务服务器的IBM Enterprise Cobol兼容。在X86和云环境中为Linux开发的战略性部署COBOL / CICS应用程序需要最小的定制工作和交货时间。此解决方案还提供了灵活性地将工作负载移动回IBM Z的组织，如果性能和吞吐量需求增加，或使用CICS事务服务器共享Z / O的业务逻辑和数据。

此外，X86上的COBOL也作为云11上的IBM CICS TX的一部分运送，使用户能够在云环境上创建，维护和部署任务和部署任务关键词的业务CICS COBOL应用程序。

支持COBOL 1985标准功能，然后选择COBOL 2002和2014标准功能。 X86上的COBOL for Linux支持以下所有所需的模块：

兼容Enterprise Cobol for z / OS和AIX的COBOL。 COBOL在X86上为Linux提供编译器选项和COBOL为AIX和Expertor Cobol提供的COBOL语言功能兼容，用于z / OS，以便在各种IBM COBOL编译器和平台之间轻松实现COBOL应用程序的可移植性。 COBOL for Linux还包含编译器选项，可控制二进制，浮点和国家数据项的endian表示。

包括源转换实用程序（SCU），以帮助将COBOL应用程序迁移到X86上Linux的COBOL。 SCU是一个独立的程序，它有助于将COBOL源程序从非IBM或自由格式源格式转换为可以在X86上由COBOL编译的格式。

提供Unicode支持，以开发使用各种国家语言的COBOL程序，并直接处理UNICODE数据。 COBOL for Linux在运行时支持Unicode UTF-16作为国家字符数据。 UTF-16是一个固定宽度Unicode编码，它提供了一种对编码纯文本的一致和有效的方法。国家（UTF-16）数据项，全国文字，内在职能，比喻常量和国家团体为开发与各种国家语言合作的COBOL计划的支持提供了支持。还提供了对汉字标准GB18030的支持。

支持开发COBOL程序以在CICS或DB2环境中运行。 Cobol for Linux上的X86支持与IBM TXSeries for MultiPlatforms的IBM TXSeries和Linux，UNIX和Windows的DB2。 DB2文件系统支持顺序，索引和相关文件。它提供了具有CICS的增强互操作性，使批量COBOL程序可访问存储在DB2中的CICS ESDS，KSD和RRDS文件。

提供XML支持，以使Cobol程序能够解析传入并生成传出XML消息。 XML使得编写COBOL应用程序更容易处理文档类型，作者和管理结构化信息，以及在计算系统中传输和共享结构化信息。

提供强大的文件系统支持，以使用具有顺序，相对，索引或线顺序组织的记录导向文件。 COBOL在X86上的Linux支持包括DB2关系数据库，CICS结构文件服务器（SFS），行顺序（LSQ），排队顺序访问方法（QSAM），记录顺序分隔（RSD），标准语言（STL）和虚拟存储访问方法（VSAM）文件系统。

模拟生成数据组（GDGS）以简化多个版本的相关数据的处理。 COBOL在X86上为Linux模拟生成数据组，时间按时间顺序相关文件，提供与Enterprise Cobol for Z / OS的更好兼容性。

支持COBOL和C / C ++之间的互列通信。介入通信支持支持从C / C ++中编写的组件例程构建COBOL应用程序。

提供扩展的源格式，允许源文本的长度为高达252字节的每行。 COBOL在X86上为Linux支持固定源格式和扩展源格式。固定源格式由文本组成，这些文本长度为每行的72个字节。扩展源格式由文本组成，长度为每行高达252个字节的文本。

X86上的COBOL可以帮助现代化，集成和管理现有的应用程序，数据和技能，以便将组织的转变为更灵活的业务。要将业务组件与供应商，合作伙伴，员工和客户联系起来，并将组织能够快速利用机会并实时响应挑战，COBOL在X86上进行Linux，可以帮助满足这些挑战，并实现现有的COBOL代码使用最新技术升级应用程序。

在开发COBOL应用程序的所有系统以及部署和执行应用程序的位置，都需要在X86上进行X86的COBOL。在X86 1.1上的COBOL在虚拟处理器核心（VPC）定价度量下许可。 VPC是一种测量单位，用于确定IBM产品的许可成本。它基于产品可用的虚拟核心（VCPU）的数量。 VCPU是一个虚拟核心，如果服务器未为虚拟机划分为虚拟机或物理处理器核心。在不同的虚拟机或不同的物理处理器核上运行的COBOL应用程序比用于开发的一个不同的物理处理器核心，需要在X86 1.1上安装COBOL，以便安装和为该机器上的产品提供的VCPU购买许可证。
