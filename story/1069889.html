谷歌发布了一项新的企业 API 政策，承诺 API 将在未来很长一段时间内仍然可用，旨在摆脱杀死服务的声誉

2021-07-27 05:22:15

谷歌云周一向现有和潜在客户承诺，围绕其核心技术开展业务是安全的，这是其从工程游乐场向真正的企业技术供应商转型的又一步。从周一开始，谷歌将把整个公司的 API 子集指定为谷歌企业 API，包括来自谷歌云、谷歌工作空间和谷歌地图的 API。根据 Google Cloud 副总裁 Kripa Krishnan 的说法，为此类别选择的 API 将包括“大部分”Google Cloud API，将遵守有关可能影响围绕这些 API 构建的客户软件的任何更改的严格准则。 Krishnan 说：“它建立在这样一个原则之上，只要客户积极使用它，就不得以向后不兼容的方式删除或更改任何功能。” “如果 API 的弃用或重大更改是不可避免的，那么我们的责任是让我们的客户尽可能轻松和轻松地体验。”该公告明确承认谷歌云客户的广泛反馈，以及互联网几个角落对谷歌在没有充分通知或预见的情况下终止对其 API 的支持的历史声誉的直接嘲笑。典型的例子可能是该公司在 2013 年决定关闭谷歌阅读器，而这只是几个月的通知，这导致了今天持续存在的大量批评。但是，虽然停止像 Reader 这样面向消费者的免费服务是一回事，谷歌认为这种服务的使用范围不够广泛，无法证明持续支持的合理性，但对付费企业客户采取这种立场则是另一回事。即使他们是少数使用特定服务的客户之一，云客户也需要知道该服务将在很长一段时间内可用且稳定。 “我们正在努力在我们的产品中不留死角，不让任何客户掉队，即使这给我们增加了大量成本，”克里希南说。当被问及她是否熟悉由 Cody Odgen 运营的“被谷歌杀死”网站和 Twitter 账户时，克里希南忍不住笑了起来。

“从互联网上的许多来源来看，我们很明显表现不佳，”她承认道。在过去的几年里，谷歌云一直试图摆脱作为一个工程驱动组织的良好声誉，该组织认为自己是网络规模基础设施计算的最重要权威，而不管其客户真正想用它的工具做什么。这种近乎傲慢的心态与 AWS 等竞争对手脱颖而出，后者凭借对云客户的早期承诺赢得了开发人员和 CIO 的信任，以及在过去几年中与地球上几乎所有公司都建立了业务关系的微软几十年。这种心态在 2019 年初开始改变，因为首席执行官 Thomas Kurian 从甲骨文公司被聘请来教授 Google Cloud 如何成为一家企业技术供应商。 Kurian 聘请了大量企业销售人员来与云购买者建立更密切的关系，并开始将 Google Cloud 的产品开发文化引导到更谦虚的姿态。 “骄傲是粗心大意的陷阱，它让许多谷歌团队认为他们的决定总是正确的，而且正确性（通过一些模糊的模糊定义）比以客户为中心更重要，”前任史蒂夫·耶格写道。谷歌和亚马逊的软件工程师，去年 8 月在一篇史诗般的帖子中谴责谷歌支持其工具的方法。 “这并不是说我们之前没有 [弃用] 政策，它只是在规模上对我们不起作用。当你还小的时候，它工作得更好，而且你已经包含了你每天与之互动的客户单位或用户，“ 她说。 “它绝对不能在云规模下工作，所以我们不得不重新考虑它。”根据新的 Google Enterprise API 政策，该公司承诺不会杀死或更改其客户“积极使用”的 API，尽管尚不清楚“积极使用”是如何定义的。如果 Google 决定需要弃用或进行更改以迫使客户对其自己的软件进行重大更改，它将至少提前一年通知即将发生的更改。新计划应该会消除云购买者可能对谷歌提出的一些异议，但谷歌在该计划下对其 API 进行更改的频率将与 AWS 和微软的类似决定进行审查。行业观察家认为，与谷歌相比，这两家领先的云提供商在过去几年中对其服务所做的更改要少得多。

云基础设施计算处于采用周期的后期阶段，去年大流行期间疯狂购买云服务的公司往往比云的早期采用者更规避风险。新的 API 政策还将让当前的 Google Cloud 客户更加放心，如果 Google 决定不再支持某项服务，他们将不必在几年后重复迁移到云所需的所有工作这对他们的业务至关重要。 “这些原则是一个更深层次的结构，真正影响了我们在 Google Cloud 中工作的方式，”Krishnan 说。 “随着我们越来越多地转向为我们的客户做正确的事，这确实是组织思维方式的转变。”
