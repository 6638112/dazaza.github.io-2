艺术家们，摆脱冒名顶替综合症——科技世界需要你

2021-07-23 04:13:35

我一生都是一名艺术家，形式多样——童书插画家、纹身艺术家，以及品牌图像和角色的制作者。多年的正式绘画培训使我具备了在这些创意口袋中交付所需的灵活性，但当它为另一个让我做好准备时，我真的感到惊讶：UI 设计。作为一名更“传统”的艺术家，UI 设计似乎是另一个领域——开发人员和图形学校的毕业生会在这里闲逛。当我寻求与图画书出版商的合作伙伴关系并在纹身店当学徒时，它从未引起我的注意。我更倾向于考虑石膏而不是线框图。只有在大流行来袭之后——封锁让我的纹身艺术家职业生涯陷入严重停顿——我才开始看到它的吸引力。我一直在为 MagicBell（一个可嵌入的通知系统）做插图工作大约一年。创始人 Hana Mohan 热衷于随附带有原创艺术作品的博客，这些作品延续了公司的独特风格。但是当她开始将注意力转向微调平台的用户界面时——并且知道我花了一些时间探索 UI 设计原则——她问我是否想尝试在 MagicBell 的应用程序上使用我新获得的知识。我有些惶恐。我不知道从哪里开始，但对我以前自学过新技能的优点说是。哈娜相信我能做到，所以我也选择了相信。我的自我教育不是从浪漫开始的，而是从一些基本的“操作方法”谷歌搜索开始的。一周后，我开始了我的第一个项目：为 MagicBell 的黑暗模式设置重新设计当前的 UI。除了为用户提供另一个视觉选项外，它还用作辅助功能设置。默认情况下，我更多地以艺术家而非设计师的身份接触这个项目——这正是为其成功铺平道路的原因。在 Google 和 Apple 的暗模式应用官方指南中找到了大量资源后，我意识到自己处于优势：挑战不在于技术要求，而在于艺术概念……我已经掌握的那些.令我惊讶的是，在我的设计生涯中，我比我意识到的要走得更远。对于现在正在挠头的正在考虑 UI 设计的艺术家们：暗模式是一种设置，它以较浅的文本呈现暗（灰色）背景。谷歌“黑暗模式流行”，你会发现一些相反的想法（它正在成为应用程序中的主要设置......但它是一种时尚。它使事情更清晰......但实际上这是一个神话。或者是吗？谁知道.) 对暗模式的主要抱怨是，许多寻求视觉“流行”的设计师将对比度设置得太高，这会使眼睛疲劳。如果没有对艺术基础进行更深入的训练，很容易犯的错误是设计出感觉笨重、功能性多于审美的东西。我觉得我可以将功能与美术相结合，所以我开始遵循古典绘画的原则：

在这里，艺术家使用的是老式的黑暗模式（使用我在 MagicBell 的下拉菜单中使用的相同阴影技术来创建透视图——但我稍后会讲到）。在这幅画的某些区域，他改变了颜色的不透明度，这样观众就会被鲜明的对比所淹没。我将这个概念应用于字体的不同不透明度。虽然我有一个预先选择的调色板，但我可以使用艺术家处理水彩画的相同原则：浓稠的、未稀释的颜色搭配更淡化、更容易上眼的颜料，仍然提供一种生动感黑暗。 （在这种情况下，我将紫色品牌颜色设置为 80% 的不透明度，与深灰色背景形成鲜明对比但又柔和。）随着我继续了解将这两个世界结合在一起的更多信息，我看到了非艺术家的诱惑在黑暗模式下选择强烈的对比色。在这里，我使用透明颜色来减弱字体、下拉菜单和顶部导航菜单中的对比度，并添加了粉红色的笔触以使其看起来更具吸引力。我还在菜单周围添加了阴影，这是直接根据我的绘画经验做出的决定——它实现了透视的错觉。为什么这在 UI 中很重要？因为艺术家在光线、颜色和视角方面的选择会引导用户的注意力，并告诉我们什么比其他需要眼球房地产的项目更重要。是的，即使在我们的应用程序中，颜色和光线也决定了我们在其中导航的简单性（或复杂性）。我对 MagicBell 的黑暗模式设置所做的决定是我在绘制它时会做出的决定。例如，当用户打开顶角的下拉菜单时，我使用阴影将其从平面对象转换为看起来更接近他们的对象。它很微妙，但巧妙地给出了透视的 3D 错觉。开始从事 UI 设计的艺术家会立即应用他们以前多次使用过的核心原则：如何组合、明暗空间、色彩理论、色彩分层、透视和其他美术框架。虽然正式培训总是有帮助，但互联网拥有自学 UI 设计所需的一切（我就是活生生的证明）。在网上，我看到越来越多的艺术家社区正在探索 UI 设计——这可能是由于大流行期间需要远程工作造成的。

观看一些 YouTube 教程，弄湿你的脚（和数字画笔）。我最喜欢的是 DevEd、Figma 和 Laith Wallace。交互设计是一个很棒的在线课程平台，专门针对 UX 和 UI。它需要预先每年订阅才能访问（200 美元），但他们的证书在业界得到广泛认可。用户体验法则提供用户体验和用户界面原则的视觉和交互表示，并可以帮助新用户界面设计师了解最佳实践。非常熟悉 Apple 和 Google 的 UI 设计指南，您可以通过 materialdesign.io 获得这些指南。这是我作为一个真正的初学者的主要研究来源（这是我学习黑暗模式基础的地方）。 TopTal 是另一个很棒的 DIY 资源，也是一个一站式商店，提供尝试新技能的演出机会。我也从 Reddit 学到了很多东西！ UI subreddit 有一个参与社区，他们总是乐于分享反馈（并指出错误）。除了易于学习和已经拥有经验的优势，艺术家转向 UI 设计师将体验到另一个好处：他们自己的复兴。

自从开始学习这项新技能后，我重新发现了驱使我成为一名艺术家的兴奋。我感到有动力去学习更多，为应用我所学的知识感到自豪，并为知道成千上万的人正在与我创造的东西互动而感到满足。随着我作为 MagicBell 设计实习生的 UI 技能不断提高，我计划扩展到 UX，并最终将艺术性和功能性巧妙地结合在一起——而不必牺牲任何一个。我带着恐惧进入了这个转变，我带着持续和充满希望的好​​奇心走出了另一边。对于寻求职业转变的艺术家来说，我保证 UI 设计！开始 获取演示并了解如何在一小时内将 MagicBell 的通知收件箱添加到您的产品中。
