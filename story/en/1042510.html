Withrawing GitHub-Backup
撤消GitHub-Backup

2020-12-30 19:43:17

I am no longer maintaining github-backup.I'll contine hosting its website and git repo for the time being, but it needsa new maintainer if it's going to survive.

我不再维护github-backup。我暂时将其网站和git repo托管在主机上，但要生存下去，它需要一个新的维护者。

I don't really think it needs to survive. If the farce of youtube-dl beingremoved from github, thus losing access to all its issues and pullrequests, taught us anything, it's that having that happen does not makemany people reconsider their dependence on github. (Not even youtube-dl itturns out, which is back on there.) Clearly people don't generallyhave any interest in backing that stuff up.

我真的不认为它需要生存。如果将youtube-dl的闹剧从github上删除，从而失去了访问其所有问题和pullrequests的权限，给我们带来了任何启示，那么发生这种情况并不会使很多人重新考虑对github的依赖。 （结果竟然没有youtube-dl，它又回来了。）显然，人们通常对备份这些东西没有任何兴趣。

As far as the git repositories on Github, they are getting archived veryeffectively by softwareheritage.orgwhich vaccumes up all git repositories from Github. Which points to aproblem, because the same can't be said for git repositories not hosted onGithub. There's a form to submit thembut the submissions often get hung up needing manual review, and it doesn'tseem to pull in new commits actively if at all, based on the few gitrepositories I've had archived there so far.

至于Github上的git存储库，它们由softwareheritage.org进行了非常有效的归档，该软件从Github中抽出了所有git存储库。这指向一个问题，因为对于不在Github上托管的git存储库，不能说相同。有一种表单可以提交，但是提交经常会被挂起来，需要人工审核，并且基于我在那里存档的几个gitrepository，它似乎也不会主动引入新的提交。远。

That seems like something it might be worth building some software to manage.But it's also just another case of Github's mass bending reality around it;the average Github user doesn't care about this and still gets archived;the average self-hosting git user may care about this slightly more, but mostwon't get archived, even if that software did get built.

似乎值得构建一些软件来进行管理，但这只是Github围绕其进行大规模弯曲现实的另一种情况; Github的普通用户仍然对此并不关心被存档；一般的自托管git用户可能会对此稍加关心，但大多数情况下都不会存档，即使该软件确实已构建也是如此。
