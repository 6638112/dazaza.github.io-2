基于具有基于NVMe的本地SSD存储的AWS Graviton2的Amazon EC2实例

2020-07-28 13:38:53

几周前，我写了一篇文章宣布新的AWS Graviton2 Amazon Elastic Compute Cloud(EC2)实例类型M6g。从那时起，数百名客户观察到了显著的性价比优势。这些公司包括Honeycom.io、SmugMug、Redbox和Valnet Inc.。

6月11日，我们宣布了两个基于AWS Graviton2处理器的新实例系列：C6g和R6g，以及现有的M6g。M系列实例旨在解决各种通用工作负载，例如应用程序服务器、游戏服务器、中型数据库、高速缓存队列、web层等。C系列实例非常适合计算密集型工作负载，例如高性能计算(HPC)、批处理、广告服务、视频编码、游戏、科学建模、分布式分析和基于CPU的机器学习推理。R系列实例非常适合内存密集型工作负载，例如开源数据库、内存缓存或实时大数据分析。要详细了解这些实例类型以及它们为何与您的工作负载相关，您可以在此视频中了解AWS副总裁兼杰出工程师James Hamilton对它们的更多信息。

今天，我有更多的消息要与大家分享：我们正在为所有三个家庭增加一个“d”变体。M6gd、C6gd和R6gd实例类型具有NVM Express(NVMe)本地连接的SSD驱动器，容量高达2 x 1.9 TB。与M5d、C5d和R5d实例相比，它们提供的存储GB/vCPU多50%。它们非常适合需要访问高速、低延迟本地存储的应用程序，包括那些需要临时存储数据以用于暂存空间、临时文件和缓存的应用程序。实例存储卷上的数据仅在关联EC2实例的生命周期内持续存在。

与同等的基于x86的M5、C5和R5系列相比，基于Graviton2处理器的实例类型提供高达40%的性价比。

所有基于AWS Graviton2的实例都构建在AWS Nitro系统之上，AWS Nitro系统是一组由AWS设计的硬件和软件，允许提供隔离的多租户、专用网络和快速本地存储。这些实例提供高达19 Gbps的Amazon Elastic Block Store(EBS)带宽和高达25 Gbps的网络带宽。

合作伙伴生态系统对这些基于ARM的AWS Graviton2实例的支持非常强大，从Linux发行版(Amazon Linux2、CentOS、Debian、Fedora、Red Hat Enterprise Linux、SuSE Linux Enterprise Server、Ubuntu或FreeBSD)到语言运行时(Java with Amazon Corretto、Node.js、Python、Go、…)。、容器服务(Docker、Amazon Elastic Container Service、Amazon Elastic Kubernetes Service、Amazon Elastic Container Registry)、代理(Amazon CloudWatch、AWS Systems Manager、Amazon Inspector)、开发人员工具(AWS Code Suite、Jenkins、GitLab、Chef、Drone.io、Travis CI)和安全与安全监控解决方案(如Datadog、Dynatrace、CrowdStrike、Qualys、Rapid7、Tenable或Honeycomb。

这些M6g、C6g和R6g系列实例现在可在以下AWS区域向您提供：美国东部(弗吉尼亚州北部)、美国西部(俄勒冈州)、美国东部(俄亥俄州)、亚太地区(东京)、欧洲(法兰克福)和欧洲(爱尔兰)。其基于磁盘的变种M6gd、C6gd和R6gd在美国东部(弗吉尼亚州北部)、美国西部(俄勒冈州)、美国东部(俄亥俄州)和欧洲(爱尔兰)AWS地区发售。
