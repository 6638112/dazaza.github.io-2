一个有趣的浏览器列表

2021-07-25 16:39:11

有每个人都听说过的浏览器。还有一些，无论出于何种原因，都消失了。这些是后者。或者，您可能知道名单上有一对夫妇。看看我能不能难倒你。 Samba 也称为 MacWWW，是 Web 创始团队的两名成员从 CERN 发布的最后一款浏览器。 Nicola Pellow 在实习期间编写了具有影响力的跨平台行模式浏览器，在完成学位后返回 CERN 继续她的工作。罗伯特·卡利奥 (Robert Calliau) 加入了她的行列，他与蒂姆·伯纳斯·李爵士 (Sir Tim Berners-Lee) 合作制定了网络的最初提案，并主要负责该提案被 CERN 采纳。他还是有史以来第一次网络会议的组织者。 Calliau 和 Pellow 认识到早期网络浏览器的一个差距。没有一个可用于 Mac。因此，他们创建了一个支持基本 HTML 的 Mac 兼容、纯文本浏览器。它总是比其他任何东西都更重要的是作为概念证明，一种向 Mac 用户展示网络的方式。即便如此，它有一个非常独特的功能，称为“跟踪”，它让浏览器的用户看到网页呈现方式的时间线。尽管跟踪对 Web 程序员很有用，但它表明这是开发人员的浏览器，而不是消费者的浏览器。在 Samba 发布后的一年内，Mosaic 将发布他们自己的 Mac 浏览器版本，由 Aleks Totić 开发，而 Samba 将不再有用。 1993 年，Marc Andreessen 离开 NCSA 的 Mosaic 项目，与 Jim Clark 一起启动他的“马赛克杀手”浏览器后，NCSA 决定开始许可他们的浏览器用于商业用途（拒绝直接进入市场是 Andreessen离开）。该许可使 Mosaic 成为了 Netscape 之前的一些最成功浏览器的起点，其中可能包括所有最成功的浏览器，Microsoft 的 Internet Explorer。其中一个项目被称为 Internet in a Box。 1994 年，O'Reilly 与网络开发领导者 Spry 合作创建了他们的 Internet in a Box 或 IBox，它将调制解调器、订阅 Internet 访问和用于连接 Internet 的软件捆绑到一个包中。 Internet in a Box 的用户可以按照一些说明立即开始浏览网页，而这需要一两年前的专门时间和一些技术知识。对许多人来说，Ibox 是他们进入 Internet 的第一步，也是最有用的部分，即网络。为了服务这些网络用户，IBox 附带了自己的浏览器。一个名为 AirMosaic 的浏览器。

Mosaic 是由研究实验室的学生构建的，因此 AirMosaic 扩展了浏览器，使其更易于日常消费者使用。今天感觉司空见惯的功能，书签和详细的浏览历史，当它们被添加到 AirMosaic 时仍然是一个新奇事物。它有一个“信息亭”模式，它删除了网站周围的所有镶边（地址栏、后退按钮、书签），只留下网站本身。这是一个您仍然可以在浏览器中找到的功能，尽管它已经在很大程度上下降了。在早期的网络时代，它非常受欢迎，当时员工仍然不得不租用大屏幕的会议室来向他们的老板演示网络的功能。 AirMosaic 是真正意义上的实用浏览器，它适合所有人，适合最新的初学者。 Dave Raggett 是 Web 标准过程中的固定工具。自 1995 年以来，他一直在 W3C 从事开放标准方面的工作。在此之前，Raggett 在惠普公司担任研究员。在他任职期间，他第一次发现了网络。随着 Web 的发展，他担心浏览器开始将标准抛在后面，在 HTML 中实现非标准功能以适应 Web 开发人员更高级的布局和设计选项。他开始着手制定新的 HTML 规范（称为 HTML+），其中包括这些非标准功能，例如表格、图形和图像的标记以及新的 HTML 标签。为了演示这些功能，他创建了一个名为 Arena 的浏览器。 Arena 在 1993 年发布时是唯一支持非官方 HTML+ 标准的浏览器。这使得它相当先进，但也不过是一个实验。 HTML+ 正式并入 HTML 后，Arena 成为 W3C 测试新 Web 标准的首选浏览器，但从未流行使用。 InternetWorks 偶尔会被提及为第一个具有标签的浏览器，该功能在浏览器大战后 Firefox 发布之前不会重新出现。然而，由 BookLink Technologies 开发的 InternetWorks 在 1990 年代初发布时捆绑了许多有用的功能，前提是您是 Windows 用户。由于采用了渐进式加载技术，该技术首先呈现文本，然后呈现图像（在宽带出现之前的日子里很重要），该浏览器以异常快速而闻名。它还建立在可移植性原则上，可以嵌入电子邮件客户端甚至 Microsoft Word 中，以便在工作时随身携带网络。但它最显着的特点——当然除了标签式浏览——是它的卡片目录。借鉴图书馆的比喻，InternetWorks 允许其用户按主题或他们想要使用的任何其他层次结构将书签组织到不同的目录中。页面可以拖放到任何目录中，在那里它们立即被保存以供离线浏览。存储大脑碎片的一站式商店。 InternetWorks 因自身的成功而受到影响。它于 1995 年在网络历史的早期被 AOL 收购，并嵌入到 AOL 平台中。几年后，AOL 以 40 亿美元收购 Netscape，使之前对 BookLink 和 InternetWorks 的投资黯然失色。

WebExplorer 可能只是这个列表中评价最高的浏览器，但也是最不为人知的。它是由 IBM 为其曾经流行但现已停产的操作系统 OS/2 创建的。该操作系统的第 3 版被称为“Warp”，它与 WebExplorer 浏览器捆绑在一起。没有多少人使用它，但那些使用它的人经常对它的功能赞不绝口。 WebExplorer 是最早支持 HTML 表格和 GIF 的浏览器之一。它内置了有用的工具，例如演示模式（很像 AirMosaic），它从浏览器中删除了 chrome，只在屏幕上留下网站。但与 InternetWorks 非常相似的是，WebExplorer 以创新的浏览历史方法脱颖而出。 1995 年 WebExplorer 发布后，网络仍然是（顾名思义）探索的地方。由于可供浏览的网站少得多，许多网络用户发现自己经常浏览以前访问过的网站。 WebExplorer 具有独特的“WebMap”，这是您访问过的网站的视觉轨迹，直接在浏览器中以分层、链接的视觉方式排列。它的界面类似于任何操作系统上的文件系统，点击一个网站会告诉你你接下来要去哪里，那些页面带你去哪里，以及你在网络空间中规划自己的方式。 WebExplorer 出现在杂志和书籍中，并被誉为浏览器中的佼佼者……大约一年了。当 Netscape 获得成功时，WebExplorer 将成为一种记忆。在 Netscape 输掉浏览器战争并将他们的浏览器作为开源发布后，一些程序员开始在 Mac 版本上工作。他们从最终成为 Mozilla Firefox 的正在开发的浏览器中获取渲染引擎，并将其调整为原生 Mac Cocoa API。结果是 Chimera，后来更名为 Camino，它在 2002 年推出了第一个测试版，比苹果团队的第一个官方浏览器 Safari 上市前一年。 Camino 在其开发周期中遇到了与任何浏览器一样多的故障。最初的团队成员之一 Dave Hyatt 离开并加入了 Safari 团队。当 AOL 接管 Netscape 时，他们在计划在 MacWorld Expo 上线前几个月剥夺了该项目的所有成员，只剩下一个基本的工作人员。结果，直到 2006 年，也就是最初开始开发四年后，它才发布了 1.0 版。尽管如此，卡米诺还是设法获得了一小部分忠实的追随者，他们拥护其在开源技术方面的血统以及对速度和性能的追求高于一切。专为 Mac 构建，它通常能够在事后排除移植到 Mac 的浏览器。而且因为它是公开构建的，所以它经常采用新功能或尖端的 UI 改进（早期版本的浏览器在单独的选项卡中打开新链接而不是新窗口，这是一个简单但非常受欢迎的功能），使其领先于它的竞争。

Camino 设法继续开发直到 2012 年左右，并在 2013 年正式停止使用浏览器。到那时，Webkit 衍生的新贵 Safari 和谷歌 Chrome 几乎让它们尘埃落定。每月两次发送关于网络历史、构建它的不可思议的人以及您从未听说过的所有网站、代码和浏览器的快讯。
