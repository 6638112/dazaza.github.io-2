Django 3.2 Released
Django 3.2发布

2021-04-06 18:50:34

This version has been designated as a long-term support (LTS) release, whichmeans that security and data loss fixes will be applied for at least the nextthree years. It will also receive fixes for crashing bugs, major functionalitybugs in newly-introduced features, and regressions from older versions ofDjango for the next eight months until December 2021.

此版本已被指定为长期支持（LTS）发布，该版本至少将应用安全性和数据丢失修复，至少将申请第三年。它还将接收用于崩溃错误，新介绍的功能的主要功能预备，以及从未来八个月到2021年12月的旧版本的回归。

As always, the release notes cover the mezcla ofnew features in detail, but a few highlights are:

一如既往，发行说明详细介绍了Mezcla的功能，但少灯是：

Customizing the type of auto-created primary keysbegins a process of migrating to BigAutoField primary key fields bydefault.

自定义自动创建的主键类型的类型迁移到BigaUtoField主键字段ByDefault。

You can get Django 3.2 from our downloads page or from thePython Package Index. The PGP keyID used for this release is Carlton Gibson: E17DF5C82B4F9D00.

您可以从下载页面或来自ThePython Package索引中获取Django 3.2。用于此版本的PGP键合是Carlton Gibson：E17DF5C82B4F9D00。

With the release of Django 3.2, Django 3.1 has reached the end of mainstreamsupport. The final minor bug fix release, 3.1.8, was issued today.Django 3.1 will receive security and data loss fixes until December 2021. Allusers are encouraged to upgrade before then to continue receiving fixes forsecurity issues.

随着Django 3.2的释放，Django 3.1已到达主流化的终点。今天发布的最终次要错误修复版本3.1.8.Django 3.1将获得安全和数据丢失修复，直到2021年12月。我们之前鼓励升级者升级，然后继续收到修复修复的修复问题。

Django 3.0 has reached the end of extended support. The final security release(3.0.14) was issued today. All Django 3.0 users are encouraged to upgrade to Django 3.1or later.

Django 3.0已达到扩展支持结束。最终的安全发布（3.0.14）今天发布。所有Django 3.0用户都会被鼓励升级到Django 3.1以后。
